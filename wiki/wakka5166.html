Примеры работ по переносу карт


1. Пример переноса карты в состав другого лота.

Например, что бы перенести определенную карточку из одного лота в другой, необходимо выполнить следующее:


1. В форме «Lots & Cards» раздела «Cards» выбираем лот – card-test, из которого необходимо перенести карточку;
2. В первом поле, напротив номеров карт, ставим галочку напротив той карточки – «al-7», которую необходимо перенести;
3. В секции «Transfer cards» в поле «Select Lot» выбираем лот, в который необходимо перенести карточку – IPS_LOT;
4. Нажимаем кнопку «Transfer cards to a selected lot» для подтверждения переноса карточки.

2. Пример переноса группы карт в состав другого лота.

Перенести группу карточек в состав другого лота можно двумя способами:


• Выделив группу карточек вручную, проставив галочки в первом поле напротив номеров карт, необходимых для переноса;
• Получив группу карточек по критериям поиска (описание параметров поиска см. «Форма “Lots & Cards”») для последующего переноса в состав другого лота.


Перенос группы карточек вручную:


1. В форме «Lots & Cards» раздела «Cards» выбираем лот – card-test, из которого необходимо перенести группу карточек;
2. В первом поле, напротив номеров карт, ставим галочки напротив тех карточек («al-201» – «al-210»), которые необходимо перенести;
3. В секции «Transfer cards» в поле «Select Lot» выбираем лот, в который необходимо перенести группу карточек – IPS_TEST;
4. Нажимаем кнопку «Transfer cards to a selected lot» для подтверждения переноса группы карточек.


См. рис.:



Перенос группы карточек, полученной по критериям поиска:


1. В форме «Lots & Cards» раздела «Cards» выбираем лот – card-test, в котором необходимо найти группу карточек по диапазону «al-17» – «al-25»;
2. В секции «Filter» выбираем значение «Card number range»;
3. По выбранному значению «Card number range» ставим параметр «Range»;
4. Вводим названия карт (начальное и конечное) для диапазона карт, которые необходимо найти;
5. Нажимаем кнопку «Find» для осуществления поиска по указанному диапазону;
6. В секции «Transfer cards» в поле «Select Lot» выбираем лот – «IPS_LOT», в который будем переносить выбранную группу карт;
7. В секции «Transfer cards» в поле «Select cards» выбираем значение «Filtered cards»;
8. Нажимаем кнопку «Transfer cards to a selected lot» для подтверждения переноса группы карточек.


См. рис.:



3. Импорт карт в AMBS

Импортировать карты в базу системы можно следующим образом:


1. В форме «Lots & Cards» раздела «Cards» выбираем лот – «card-import», в который необходимо импортировать карточки.
2. Нажимаем кнопку «Import cards» в секции по созданию и редактированию карточных лотов.
3. В открывшейся форме по импорту карточек «Card importation» указываем путь к файлу с данными по карточкам – import.csv


Примечание: Файл «import.csv» выглядит следующим образом, см. рис.:




При импорте файла, поля карт могут принимать следующие значения:


ПолеНазначение
Card numberНомер карты. Может содержать любые знаки, буквы и цифры, кроме кавычек. Длина поля составляет 32 символа.

TypeТип авторизации карт. Может принимать следующие значения:
«0» – PIN – авторизация по пин-коду карты;
«1» – ANI – авторизация карты по номеру звонящего абонента;
«2» – PIN+ANI – смешанная авторизация по пин-коду и номеру абонента;
«3» – Wi-Fi – авторизация для Wi-Fi-карт.

CategoryТип карты, может принимать значения:
«1» – Debit – предоплаченные карты;
«2» – Credit – кредитные карты.

StatusСтатус карты:
«0» – new – не активированные карты;
«1» – active – активированные карты;
«2» – blocked – заблокированные карты;
«3» – closed – закрытые карты.


4. Нажимаем кнопку «Ok» для подтверждения загрузки файла с данными по карточкам.


См. рис.:




Примечание: Поле «Regenerate certificates» отвечает за генерацию сертификатов при импорте Wi-Fi-карт. Может принимать следующие значения:
«Параметр включен (стоит галочка)» – генерация сертификатов Wi-Fi-карточек при импорте в систему;
«Параметр выключен (нет галочки)» – импорт Wi-Fi-карточек без генерации сертификатов.


Пример импорта карт

Требуется добавить в систему 3 карты:


CARD ID 12189 PIN 40304236 BALANC 1.21
CARD ID 13170 PIN 71094413 BALANC 17.2
CARD ID 14161 PIN 70320579 BALANC 20


С учетом следующих условий:

  1. Карты нужно добавить в отдельный лот с именем – Test.
  2. Период активации карточек должен быть неограниченным.
  3. Карточки после активации должны работать только 90 дней.

Решение:
1. Заводим лот «Test»:

  • в поле «Lot prefix» – указываем префикс лота test (обязательное поле);
  • в поле «Pin length» – ставим значение 5 символов
(для каждой карты в системе должны быть ПИН+КЕЙ, поскольку в исходных данных ПИН дан одним набором из 8 цифр,
прийдется его разделить на ПИН+КЕЙ в Excel-е, например как 5+3);
  • в поле «Pin key length» – ставим значение 3 (5+3);
  • в поле «Activation period» – ставим значение 10 лет (неограниченного срока нет, 10 лет достаточно большой период для карты );
  • в поле «Live period» – ставим 90 «дней».


Примечание: Описание полей лотов и их назначение вы можете посмотреть в разделе “Lots & Cards”
