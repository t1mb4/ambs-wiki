Схема взаимодействия системы IP Studio AMBS с карточной платформой


Наглядно, схему работы карточной платформы на базе IP Studio AMBS можно представить в следующем виде:



Как видно из схемы, принцип работы не содержит лишних операций, при этом все процессы проверки проходят через карточный модуль системы, что позволяет оператору контролировать работу и получить статистику по работе карточных лотов в любой момент времени.
Структуру карточного модуля и его связь с базовой системой IP Studio AMBS в процессе работы с карточным трафиком можно представить в виде следующей схемы:



Как видно из рисунка, карточный модуль системы напрямую взаимодействует только с сервером доступа карточной платформы, что позволяет использовать его в двух вариантах:
  1. Использовать модуль как отдельную систему по учету звонков, прошедших по карточкам и балансу карточек. При этом направляя карточный трафик напрямую, через сервер доступа на конкретного оператора;
  2. Использовать модуль как отдельную систему по учету звонков прошедших по карточкам и балансу карточек. При этом, направляя карточный трафик на программный коммутатор (например, MVTS-сервер), который будет работать под управлением базовой системы IP Studio AMBS. Далее, используя функции динамической маршрутизации системы AMBS, карточный трафик будет распределяться между терминирующими операторами с учетом факторов цены и качества направлений.

Используя второй вариант, оператор также получает возможность запустить несколько карточных платформ или совместить работу с транзитным трафиком и карточной платформой используя всего один сервер доступа. Оператор может организовать этот функционал с помощью использования префиксов.
