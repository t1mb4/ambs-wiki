Секция шлюзов, привратников и пользователей оператора


Посмотреть пример работы с шлюзами



Ниже секции сайтов оператора расположена секция для шлюзов, закрепленных за выбранным сайтом оператора.
В заголовке секции присутствует панель навигации, позволяющая просматривать один тип шлюзов за раз:



ПолеНазначение
GatewaysШлюз. Аппаратно-программный комплекс, обеспечивающий обмен данными между сетями разных типов

GatekeepersПривратник. Выполняет функции управления зоной сети H.323

Default usersАбонент сети IP телефонии (шлюз, терминал), регистрирующийся на привратнике по RAS протоколу, с использованием логина и пароля

Configured usersАбонент сети IP телефонии (шлюз, терминал), регистрирующийся на привратнике по RAS протоколу, с использованием логина и пароля и имеющий свой ip-адрес и другие параметры конфигурации


Далее в панели секции расположен фильтр “active ip”, позволяющий просматривать не обходимую информацию, неотображая всего остального.
Фильтр может принимать следующие значения:


ЗначениеОписание
Active ipПоказывает все активные на данный момент времени шлюзы на всех рабочих софтсвичах.

AllПоказывает все шлюзы закрепленные за этим сайтом, активные и закрытые.

All on working serversПоказывает все шлюзы (активные и закрытые), заведенные только на активных в данный момент времени софтсвичах.

All on closed serversПоказывает все шлюзы (активные и закрытые), заведенные только на выключенных в данный момент времени софтсвичах.




Секция шлюзов оператора (GW – Gateways)


По умолчанию всегда открывается форма шлюзов оператора (gateways).



В форме шлюзов присутствуют следующие кнопки управления:


КнопкаНазначение

Позволяет добавить новый шлюз к этому сайту оператора

Примечание: Тип добавляемого шлюза определяется текущим положением в секции шлюзов. Например, если вы просматриваете шлюзы (gateways) оператора, будет добавлен новый шлюз, если вы просматриваете привратники (gatekeepers) оператора, будет добавлен новый привратник.


Вызывает форму для редактирования параметров выбранного шлюза

Удаляет выбранный шлюз, если по нему небыло звонков в течение последних 3х месяцев.

Примечание: Мы не рекомендуем удалять из системы шлюзы, если через них проходил трафик, это повлечет за собой ошибки в статистике, в некоторых случаях эти ошибки могут быть критическими. Рекомендуем закрывать ненужные шлюзы параметром – “Closed” или “Stop Date”-ой.


При наведение на эту иконку курсора мыши, будет показано всплывающее окно с параметрами шлюза.


Форма для редактирования параметров шлюза имеет следующий вид:



Описание полей формы:


ПолеНазначение
Operator ip

IP адрес шлюза. Поле обязательно для заполнения. В систему заносятся только уникальные адреса шлюзов.

Примечание: Записи адресов типа:
111.222.133.1–255
111.222.133.44/29
работать не будут. Все подсети записываются через маску адреса.


IP MaskМаска соединений шлюза. Параметры этого шлюза будут применимы ко всем соединениям, удовлетворяющим значениям пары address/mask. Для оригинирующих операторов есть возможность указывать подсети целиком.
Значение по умолчанию: 255.255.255.255

Примеры:

192.168.0.1–255 = ip: 192.168.0.1 mask:255.255.255.0

192.168.0.98 / 32 = ip: 192.168.0.98 mask:255.255.255.255

192.168.0.98 / 29 = ip: 192.168.0.98 mask:255.255.255.248

192.168.0.98 / 28 = ip: 192.168.0.98 mask:255.255.255.240

192.168.0.98 / 26 = ip: 192.168.0.98 mask:255.255.255.192

192.168.0.98 / 24 = ip: 192.168.0.98 mask:255.255.255.0

192.168.0.98 / 23 = ip: 192.168.0.98 mask:255.255.254.0

Gateway portПорт для входящих звонков
Значение по умолчанию: 1720

Gateway IDИдентификатор (название шлюза), должен быть уникальным для каждого шлюза.
Если шлюз добавляется в систему повторно, идентификатор должен совпадать с идентификатором уже существующего в системе шлюза.

Set proxyПараметр проксирования трафика:
0 – проксирование только сигнального трафика,
1 – проксирование сигнального и медиа (речевого, мультимедийного) трафика.
Значение по умолчанию: 0

Gateway modeРежим работы шлюза:
0 (nothing) – Шлюз отключен,
1 (originate) – Оригинация. Шлюз будет использоваться только для отправки трафика на операторов,
2 (terminate) – Терминация. Шлюз будет использоваться только для приема трафика от операторов,
3 (both) – Все. Трафик будет и отправляться, и приниматься с этого шлюза.
Значение по умолчанию: 3

Gateway typeТип шлюза. (Необязательное поле)
0 – тип шлюза неизвестен,
1 – CISCO,
2 – VocalTec,
3 – Samsung,
4 – Media Ring.
Значение по умолчанию: 0

IP precedence

Содержит приоритет IP-пакетов, проходящих через шлюз. (Необязательное поле)
где номер – приоритет пакетов[0..7]
Значение по умолчанию: 3 (эквивалентно операционной системе)

Примечание: Рекомендуется назначать ip_precedence выше 3, минимально возможному числу шлюзов, так как большое количество звонков с приоритетом больше 3 может вызвать нестабильность ядра операционной системы.


CapacityМаксимальное количество звонков, которое MVTS может установить с участием данного шлюза
Значение по умолчанию: 255

Priority within operatorПриоритет, согласно которому выбирается очередность использования ip-адресов оператора.
Если у оператора несколько шлюзов, то первым будет использован тот, у которого в данном поле стоит меньшее значение

Enter dst_translate

Содержат правила преобразования номера вызываемого (dst_translate) абонента. (Необязательное поле)
Как правило, состоит из двух частей, разделенных символом “/”

  • выражение regexp, задающее строку поиска
  • строка замены

Строка поиска для удобства может быть разделена на секции символом“|”. В строке замены можно использовать подстроку, удовлетворяющую условию секции, указав номер секции после символа ”\”. Для использования в строке замены подстроки, удовлетворяющей всем секциям одновременно служит символ “&”.

Src_translateПравила преобразования исходящего номера вызывающего абонента (АОН). (Необязательное поле)
Работает по аналогии с полями трансляций входящего номера, но обрабатывает номер АОНа.
Трансляция АОН номера, которая будет отправлена дальше, т.е. имеет смысл заполнять у терминирующих операторов.
У оригинирующих операторов работать не будет, на входе АОН обрабатывается параметром “in_src_translate”.
В форме нет поля для указания правил “in_src_translate”, его можно добавить в поле “Additional parameters”.
Пример: Additional parameters: in_src_translate=.*/7495123
Все АОНы на входе заменять на номер “7495123”

In_dst_translateПоле трансляции входящего номера с задаваемого адреса оператора. (Необязательное поле)
Указывается параметр трансляции, который должен преобразовывать номер на входе.
Используется только для входящих звонков. Например, префикс, который должен отрезаться при получении звонков с этого адреса, этого оператора.

Пример 1: Терминирующий оператор просит присылать все звонки с префиксом 100#.


В этом случае поле “Enter dst_translate” у шлюза этого терминирующего оператора должно иметь значение: [0-9]*/100#&


Пример 2: Оригинирующий оператор будет присылать вам трафик с префиксом 222#.


В этом случае поле “Enter in_dst_translate” у шлюза этого оригинирующего оператора должно иметь значение: 222#|.*/\2

NAT rtpПоддержка NAT-а.
0 – сервер не за NAT-ом,
1 – сервер находится за NAT-ом.
Значение по умолчанию: 0

IP StateСтатус данного шлюза.
Closed – отключён,
Opened – работает.
Значение по умолчанию: Opened

Change siteПозволяет перенести редактируемый шлюз на любой другой сайт в системе

Select mera serverТаблица MVTS-серверов, на которых заведен данный шлюз
Значение по умолчанию: Все доступные MVTS-сервера

Additional paramsПоле для дополнительных параметров шлюза, не вошедших в таблицу параметров, но необходимых для правильной работы данного шлюза.
Например, в нем можно указать формат разрешенных/запрещенных кодеков и т.д. (Запись должна быть в формате MVTS-сервера)

Start dateДата начала работы шлюза
Значение по умолчанию: Текущее число

Stop dateДата окончания работа шлюза
Значение по умолчанию: 9999–12–31


Внизу формы расположены следующие кнопки управления:


КнопкаНазначение
Сохраняет внесенные изменения и закрывает окно редактирования

Сбрасывает внесенные изменения и закрывает окно формы редактирования

Удаляет редактируемый шлюз оператора

Примечание: Удаление возможно только, если по данному шлюзу не проходил трафик в течение последних 3-х месяцев.
