Форма “User interface”


Позволяет клиентам оператора самостоятельно зайти в систему AMBS по специальному логину и паролю и просмотреть статистику по прошедшим через их карты звонкам, текущий баланс карт и при необходимости сменить пин-ключ их карт.
Внешний вид формы представлен на следующем рисунке:



Описание полей формы:


ПолеНазначение
Card numberПозволяет ввести название карточки абонента.

PIN codeПозволяет ввести пин-код карточки абонента.

Time period Период времени, за который будут показаны звонки по выбранной карте.
Может принимать следующие значения:
Today – сегодня;
Yesterday – вчера;
3 days – три дня;
7 days – семь дней, не включая текущий день;
Month – текущий месяц;
Last month – предыдущий месяц;
Period – период, заданный вручную полями From-To.
Значение по умолчанию – Today.

Select statistic typeПозволяет указать тип карточки абонента. Может принимать значения: VoIP или Dial-up (при условии, что установлен диал-ап модуль IP Studio AMBS).
Значение по умолчанию: VoIP.

RecordsКоличество выводимых строк на дисплей ПК. Может принимать значения: 10, 30, 50, 300.
Значение по умолчанию: 10.


В форме присутствуют кнопки управления:


КнопкаНазначение
Find Осуществляет поиск карточки по введенным параметрам.

Add pay. Позволяет добавить платеж на баланс карточек.

Show pays. Показывает историю платежей на балансе карточки.


Для того чтобы получить данные по своей карте в этой форме абонент должен указать номер своей карты и пин-код доступа. При нажатии на кнопку «Find» программа выведет на экран статистику по звонкам, совершенным с использованием этой карты за весь период времени указанный в правой части формы в секции «Time period».



Из рисунка видно, что отчет по работе карты отображается в виде двух таблиц, в первой показаны общие данные по карте: дата активации карты, баланс и статус карты. Ниже будут показаны в виде таблицы все звонки, прошедшие через эту карту.


Примечание: В первой таблице, абонент при необходимости, может изменить пин-ключ к своей карте.

Для того чтобы абоненты карточной платформы имели доступ только к этой форме, пользователю следует выполнить следующие действия:


1. Завести группу пользователей «Сards» в разделе «Users» (Administration -> Users);
2. Завести одного общего пользователя в группе карточных пользователей, например «Carduser»;
3. Назначить права доступа карточной группы пользователей только для формы «User interface» в разделе форм (Administration -> Forms&Actions).


Примечание: Пример заведения группы и назначение прав на форму следует смотреть в разделе «Administration».

После этого оператору останется сообщить данные доступа своим абонентам, и они смогут самостоятельно проверять свой баланс и статистику по прошедшим звонкам.
