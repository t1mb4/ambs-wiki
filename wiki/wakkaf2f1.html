Проверка работы процессов IP Studio AMBS


На сервере(ах) c IP Studio AMBS должны быть запущены следующие процессы:


1. Радиус процессы. Все радиус-процессы можно разделить на следующие условные группы:

  • транзит(mvts),
  • транзит(cisco),
  • карточная платформа(card),
  • диал-ап(dial).

Для проверки в консоли выполнить команду:


[root]# ps ax | grep rad

Результаты будут иметь следующий вид:


Транзит(mvts):
8543? S 3:07 /usr/local/billing/bin/radiusd -config_file /usr/local/billing/radius/radius.cfg -pid_file /usr/local/billing/radius/radius.pid
8550? S 89:09 /usr/local/billing/bin/radiusd -config_file /usr/local/billing/radius/radius.cfg -pid_file /usr/local/billing/radius/radius.pid
8551? R 90:06 /usr/local/billing/bin/radiusd -config_file /usr/local/billing/radius/radius.cfg -pid_file /usr/local/billing/radius/radius.pid
8552? S 91:12 /usr/local/billing/bin/radiusd -config_file /usr/local/billing/radius/radius.cfg -pid_file /usr/local/billing/radius/radius.pid
8553? S 89:27 /usr/local/billing/bin/radiusd -config_file /usr/local/billing/radius/radius.cfg -pid_file /usr/local/billing/radius/radius.pid

Транзит(cisco):
21496? S 0:21 /usr/local/billing/bin/rad_cisco -config_file /usr/local/billing/radius/cisco.cfg -pid_file /usr/local/billing/radius/rad_cisco.pid -log_file /usr/local/billing/radius/cisco.log

Карточная платформа(card):
4111? S 0:06 /usr/local/billing/bin/radiusd -config_file /usr/local/billing/radius/card.cfg -pid_file /usr/local/billing/radius/rad_card.pid -log_file /usr/local/billing/radius/card.log

2. Демон, принимающий сигнал от IP Studio AMBS на выполнение разовых процессов и проверки алертов, должен присутствовать в виде 2х процессов (если работает 1 или 3 это уже ошибка). Чтобы проверить, нужно из консоли сервера выполнить команду:


[root]# ps ax | grep bill_sig_svr

Результат:


28763? S 0:00 /usr/local/billing/bin/bill_sig_svr.pl
29519? S 0:00 /usr/local/billing/bin/bill_sig_svr.pl

3. MySQL сервер. Чтобы проверить, нужно из консоли выполнить команду:


[root]# ps ax | grep mysql

Результат:


2811? S 0:00 /bin/sh /usr/local/billing/mysql/bin/mysqld_safe --user=ipstudio --datadir=/usr/local/billing/base --pid-file=/usr/local/billing/base/mysql.pid
2859? S 1625:16 /usr/local/billing/mysql/libexec/mysqld --basedir=/usr/local/billing/mysql --datadir=/usr/local/billing/base --user=ipstudio --pid-file=/usr/local/billing/base/mysql.pid --skip-external-locking --port=3306 --socket=/tmp/mysql.sock

4. Апач – на AMBS сервере. Чтобы проверить, нужно из консоли сервера выполнить команду:


[root]# ps ax | grep httpd

Результат (количество процессов может изменяться):


3246? S 0:14 /usr/local/billing/apache/bin/httpd
3248? S 0:01 /usr/local/billing/apache/bin/httpd
3249? S 0:01 /usr/local/billing/apache/bin/httpd
3250? S 0:01 /usr/local/billing/apache/bin/httpd
3251? S 0:01 /usr/local/billing/apache/bin/httpd
3252? S 0:01 /usr/local/billing/apache/bin/httpd
3256? S 0:01 /usr/local/billing/apache/bin/httpd
3264? S 0:01 /usr/local/billing/apache/bin/httpd
3265? S 0:01 /usr/local/billing/apache/bin/httpd

Если вы используете удаленные MVTS-сервера, на них должен постоянно выполняться скрипт для постоянного выполнения задач (каждые N мин). Чтобы проверить, нужно из консоли сервера выполнить команду:


[root]# ps ax | grep bill_sig_svr

Результат:


28763? S 0:00 /usr/local/billing/bin/bill_sig_svr.pl

Если у Вас выполняются все вышеперечисленные процессы, процесс установки выполнен успешно, и вы можете приступать к процессу конфигурации системы IP Studio AMBS.
