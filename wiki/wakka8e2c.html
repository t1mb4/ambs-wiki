Форма “Routs Test”


Форма тестов позволяет посмотреть доступные оператору (оригинатору) направления терминации.
Можно увидеть, на каких терминирующих операторов пойдет звонок оператора с учетом всех заданных правил маршрутизации и его цены на данное направление.
Внешне форма выглядит следующим образом:



Проверить прохождение кода(номера) можно двумя методами: задав точные данные по шлюзу/пользователь начавшему звонок или выбрав конкретного оператора, заведенного в системе.

Описание полей формы:

ПолеНазначение
Select siteПоле для выбора сайта оригинирующего оператора, для которого будет строиться список, доступных ему маршрутов.

Select mera serversАдрес mvts-сервера, через который будет проходить звонок.

Destination numberНомер (или код), по которому требуется посмотреть список доступных для терминации маршрутов.

Source userЛогин оригинирующего оператора.
Не обязательный параметр.
Параметр активен, только если в сайтах выбран пункт поиска по параметрам источника (Find by source parameters).

Source ipШлюз оригинирующего оператора.
Не обязательный параметр.
Параметр активен, только если в сайтах выбран пункт поиска по параметрам источника (Find by source parameters).

Source number inАОН оригинирующего оператора.
Не обязательный параметр.
Параметр активен, только если в сайтах выбран пункт поиска по параметрам источника (Find by source parameters).

Closed routes (y/n)Параметр для просмотра закрытых направлений операторов.
Существующих направлений, на которые звонок может пройти, но направления закрыты по каким-то причинам (не работающий код, неправильная цена и т.д.)

Check number lengthПараметр для проверки указанного номера в поле “Destination number” на правильность, если в системе используется фильтрация на длину номера.

DebugВыводит на экран техническую информацию по работе формы.
Обычно требуется для разбора проблем с маршрутизацией при обращении в техническую поддержку.

Route limitКоличество строк с доступными маршрутами терминации выводимых на экран.

Show routeКнопка запуска выборки. Обрабатывает запрос и выдает результат на экран.


Для того чтобы воспользоваться данной формой достаточно:

1. В поле “Select site” необходимо выбирать оригинирующего оператора.

2. В поле “Select MERA servers” надо выбрать один из MVTS-серверов, прописанных в системе (не обязательный параметр, если мера одна).

3. В поле ”Enter destination number” нужно ввести начальные цифры кода направления или сам проверяемый номер.

4. Нажать кнопку “Show route”.

В результате получим таблицу следующего вида:



Сразу под формой показана строка с информацией об оригинаторе, его направлении из тарифного плана, выбранное системой для прохождения проверяемого звонка(кода).
На показанном примере видно, что система выбрала следующие данные для прохождения звонка:
  • Цена оригинации – 100.00;
  • Код оригинации – 749562;
  • Сортировка доступных маршрутов – по «Завершенным звонкам»;
  • Используются дополнительные правила маршрутизации, заданные в ручную – Да;
  • Проверяемое направление состоит в роутинговой группе – Нет.

Далее в таблице приведены доступные направления терминации. Система покажет список с учетом всех установленных для данного направления оператора правил маршрутизации и сортировки.
Например, если у оригинирующего оператора указано сортировать доступные ему направления по цене, то направления будут строиться с учетом цены, т.е. с минимальной цены до максимальной.
В таблице доступных направлений различными цветами подсвечена доступность операторов и правила маршрутизации:

ЦветНазначение
Оператор, поставленный вручную первым выбором.
Настройки ручной маршрутизации производятся в форме “Operator route parameters”.
В строке оригинатора есть параметр “Use route parameters”, по этой ссылке можно перейти напрямую в форму маршрутизации, для просмотра правила.

Оператор, поставленный вручную вторым выбором.
Настройки ручной маршрутизации производятся в форме “Operator route parameters”.
В строке оригинатора есть параметр “Use route parameters”, по этой ссылке можно перейти напрямую в форму маршрутизации, для просмотра правила.

Оператор, поставленный вручную третьим выбором.
Настройки ручной маршрутизации производятся в форме “Operator route parameters”.
В строке оригинатора есть параметр “Use route parameters”, по этой ссылке можно перейти напрямую в форму маршрутизации, для просмотра правила.

Дорогие направления. Направления операторов, на которые данный оригинатор пройти не может из-за цены.

Закрытые направления. Данные направления операторов были закрытые вручную в правилах маршрутизации.
Закрыть направление можно или в его тарифном плане, или в форме “Operator route parameters”.

Направления, находящиеся в одной группе с направлением оригинатора.
Группировка направлений производится в форме “Route groups”.

Прямой маршрут на заданного оператора.
Жесткое указание прямого маршрута прохождения звонка, без контроля цены и правил сортировок.


Таблица со списком доступных операторов состоит из следующих полей:
ПолеНазначение
NumПорядковый номер по списку.

IDИдентификатор сайта оператора.

SiteНазвание сайта оператора.

GatewayИдентификатор шлюза оператора.

Proxy typeСтатус режима проксирования:
1 – проксирование используется
0 – проксирование не используется

Base priceЦена в базовой валюте системы.
Валюта системы по умолчанию – USD.

PriceЦена в валюте терминирующего оператора.

CurrencyВалюта терминирующего оператора на выбранное направление.
Код выбранного направления терминирующего оператора показывается в поле “Code”.

ASRПроцент успешных звонков.

ACDСредняя продолжительность разговора в минутах.

Short callsПроцент коротких звонков, у которых продолжительность меньше 10 секунд.

All callsОбщее количество прошедших звонков.

Complete callsКоличество звонков с ненулевой длительностью.

QiИндекс качества оператора по выбранному коду.
Выбранный код из тарифного плана терминирующего оператора показывается в поле “Code”.

Dst number transНомера на выходе, такой каким он уйдет к терминирующему оператору.

CodeКод из тарифного плана терминирующего оператора, который система нашла по запросу.


Примечание:
  1. Если на проверяемое направление выводится надпись “Codes not found”, это значит, что никто из терминирующих операторов не предоставляет это направление.
  2. Если в полученном листе с доступными маршрутами, у кого то из операторов отсутствует цена или код (т.е. оператор в списке показан, но поле цены или кода – пустое), это означает, что в тарифном плане этого оператора присутствует лишняя пустая запятая в одном из направлений. Проще всего исправить подобные ситуации в форме “Edit Destinations” воспользоваться кнопкой “Update all codes”. AMBS сама проверит все коды в базе и уберет все лишние запятые.
