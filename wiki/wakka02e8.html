Секция расширенных RAS-пользователей (configured users)


Для отображения формы пользователей оператора нужно выбрать пункт “configured users” из меню шлюзов оператора:



Внешний вид формы пользователей аналогичен форме шлюзов оператора. В форме пользователей присутствуют следующие кнопки управления:

КнопкаНазначение
Позволяет добавить нового пользователя оператору.

Вызывает форму для редактирования параметров выбранного пользователя.

При наведении на эту иконку курсора мыши, будет показано всплывающее окно с параметрами пользователя.


Для добавления нового пользователя или редактирования уже существующих в системе при нажатии на соответствующие кнопки будет вызвана форма параметров пользователя:



Описание полей формы:

ПолеНазначение
User nameИмя пользователя в системе. Используется для идентификации пользователя при регистрации его на MVTS и при запросах к системе удаленной аутентификации RADIUS.

User passwordПароль пользователя. Поле обязательно для заполнения. Используется системой совместно с именем пользователя для тех же целей.

User ipIP адрес RAS-пользователя.

RAS IDИдентификатор шлюза пользователя, имя должно быть уникальным для каждого шлюза.

Proxy typeПараметр проксирования трафика.
0 – проксирование только сигнального трафика,
1 – проксирование сигнального и медиа (речевого, мультимедийного) трафика.
Значение по умолчанию – 0.

Gateway typeТип шлюза.
0 – Тип шлюза неизвестен,
1 – CISCO,
2 – VocalTec,
3 – Samsung,
4 – Media Ring.
Необязательное поле.
Значение по умолчанию: 0

IP precedenceСодержит приоритет IP-пакетов, проходящих через шлюз.
Необязательное поле.
Здесь приоритет пакетов номер – [0..7]
Значение по умолчанию – 3 (эквивалентно операционной системе)
//Примечание: Рекомендуется назначать ip_precedence выше 3
минимально возможному числу шлюзов, так как большое количество звонков с приоритетом больше 3 может вызвать нестабильность ядра операционной системы.//

CapacityМаксимальное количество звонков, которое MVTS может установить с участием данного шлюза.

Priority within operatorПриоритет, согласно которому выбирается очерёдность использования ip-адресов оператора. Если у оператора несколько шлюзов, то первым будет использован тот, у которого в данном поле стоит большее число.

Dst translateСодержит правила преобразования номера вызываемого абонента. Как правило, состоит из двух частей, разделенных символом “/”
* выражение regexp, задающее строку поиска
* строка замены
Строка поиска для удобства может быть разделена на секции символом “|”. В строке замены можно использовать подстроку, удовлетворяющую условию секции, указав номер секции после символа ”\”. Для использования в строке замены подстроки, удовлетворяющей всем секциям одновременно служит символ “&”. Необязательное поле.

Src translateСодержит правила преобразования номера вызывающего абонента. Как правило, состоит из двух частей, разделенных символом “/”
* выражение regexp, задающее строку поиска
* строка замены
Строка поиска для удобства может быть разделена на секции символом “|”. В строке замены можно использовать подстроку, удовлетворяющую условию секции, указав номер секции после символа ”\”. Для использования в строке замены подстроки, удовлетворяющей всем секциям одновременно служит символ “&”. Необязательное поле.

In dst translateПредварительная трансляция, выполняемая при поступлении звонка от этого шлюза, перед вызовом поиска по объектам набора (dial peers).
Параметры полезны, когда какой-либо шлюз присылает номера не в формате Е164.

nat_rtpФлаг направления RTP пакетов. Используется при нахождении шлюза за устройством трансляции адресов (NAT).
Необязательное поле.
0 – NAT не используется,
1 – Прокси будет посылать RTP пакеты не на адреса, сообщенные шлюзом в H.245 сессии, а на адреса, с которых RTP пакеты поступают реально.
Значение по умолчанию – 0.

Start dateДата начала работы привратника.

Stop dateДата окончания работы привратника.

Ip stateСтатус данного пользователя:
closed – отключён,
opened – работает.

Select mera serversMVTS-сервера, на которых будет прописан этот привратник.
Обязательное поле.

Additional parametersПоле для дополнительных параметров привратника. Например, для формата поддерживаемых кодеков, алертов и т.д.
(Запись должна быть в формате записи MVTS-сервера).


Внизу формы расположены следующие кнопки управления:

КнопкаНазначение
Сохраняет внесенные изменения и закрывает окно редактирования

Сбрасывает внесенные изменения и закрывает окно формы редактирования
