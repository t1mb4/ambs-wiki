Форма “Calls Details”


Форма предназначена для просмотра детальной статистики по совершенным звонкам за период времени.
Внешний вид формы показан на следующем рисунке:



В форме можно указать конкретных операторов(оригинирующих, терминирующих), установить интервал времени и дополнительные параметры статистики.

Описание полей формы:

ПолеНазначение
Show closed operatorsПоказывает в списках оригинирующих и терминирующих операторов «закрытых» операторов(на данный момент неактивных).
По умолчанию данный параметр выключен.

Select originate operator and IPПозволяет выбрать для просмотра конкретного оператора, от которого пришли звонки.
После выбора оператора, можно выбрать любой из его сайтов и шлюзов, для получения более конкретной информации.
Значение по умолчанию: Any (Все операторы).

Select terminate operator and IPПозволяет выбрать для просмотра конкретного оператора, на которого ушли звонки.
После выбора оператора, можно выбрать любой из его сайтов и шлюзов, для получения более конкретной информации.
Значение по умолчанию: Any (Все операторы).

Select Mera IPЕсли используется несколько MVTS-серверов, можно получить данные по конкретному серверу, выбрав его в этом списке.
Значение по умолчанию: Any (Все сервера).

Enter destination codeФильтр по первым цифрам номера. Позволяет получить информацию по конкретному направлению/коду/номеру.

Q931 filterФильтр кодов завершения (Q931). Позволяет выбрать звонки с конкретным кодом завершения.

ShowТип звонков, показываемых на экране.
Может принимать следующие значения:
All – Показать все звонки.
Not zero calls – Показать только ненулевые звонки.
Zero calls only – Показать только нулевые звонки.
Not calculated calls – Показать не тарифицированные звонки. Звонки, для которых система не может найти тарифные планы для их тарификации.
Not calculated calls (term) – Все звонки, не тарифицированные оригинирующим оператором.
Not calculated calls (orig) – Все звонки, не тарифицированные терминирующим оператором.
Значение по умолчанию – All

Time periodПериод времени, за который будут показаны звонки.
Может принимать следующие значения:
Today – Сегодня.
Yesterday – Вчера.
3 days – Три дня.
7 days – Семь дней, не включая текущий день.
Month – Текущий месяц.
Last month – Предыдущий месяц.
Period – Период, заданный вручную полями From-To.
Last 24 hours – Период за последние 24 часа, указывается в часах
Значение по умолчанию – today

Period From-ToДаты начала и завершения расчетного периода, за который будет выбираться статистика (задаются вручную, с помощью календарей).

Time From-ToВремя в часах, начала и завершения выборки.
Может принимать значения:
full period – значениями from-to указываются час начала выборки и час завершения выборки, все дни входящие в выборку будут обработаны полностью (24 часа).
Например можно выбрать все звонки за 5 дней начиная с 10:00 в понедельник и заканчивая в 19:00 в пятницу.
per date – значениями from-to указывают час начала и завершения выборки за каждый день выборки.
Например можно выбрать все звонки за 7 дней прошедшие только днем в интервал 10:00-22:00.

Search in time zoneПостроение выборки по выбранному часовому поясу.
Значение по умолчанию: Часовой пояс системы.

E-mail toОтправить результат на выбранный почтовый ящик.
Адреса почты можно заводить в разделе “Managers” (Company-Managers).
Значение по умолчанию: Неактивен

ShowТип статистики.
Может принимать следующие значения:
Summary – Краткая (итоговая) статистика, краткий вариант. Формируется быстро.
Details – Детальная статистика, с полными выкладками по каждому звонку. Загружается дольше, чем краткая статистика.
Значение по умолчанию: summary


В форме присутствуют следующие кнопки управления:

КнопкаНазначение
Show calls from buffersПоказать статистику последних звонков,находящихся на данный момент в буфере базы данных.

Show infoПоказать статистику по звонкам с учетом заданных в форме раздела параметров.

Show CDR parametersФорма полей отображения.
Открывает окно с полями, которые будут показаны в статистике для каждого звонка.
Поля соответствуют полям cdr-файлов.



В таблице суммарной статистики показывается следующая сводная информация:

ПолеНазначение
All ringsОбщее количество звонков по этому направлению.

Successful ringsКоличество успешных звонков, у которых код возврата – 16, 17 и 19.

ASRПроцент успешных дозвонов.

Complete callsПроцент ненулевых звонков.

ACDСредняя продолжительность разговора в минутах.

Short callsПроцент коротких звонков, у которых продолжительность меньше 30 секунд.

All minutesОбщее количество минут.

Source traffic IN(Mb.)Объем трафика в мегабайтах, пришедший от оригинирующего оператора.

Source traffic OUT(Mb.)Объем трафика в мегабайтах, отправленный оригинирующему оператору.

Destin traffic IN(Mb.)Объем трафика в мегабайтах, пришедший от терминирующего оператора.

Destin traffic OUT(Mb.)Объем трафика в мегабайтах, отправленный терминирующему оператору.

SUM Terminate (у.е.)Сумма терминации за прошедший объем трафика.

SUM Originate (у.е.)Сумма оригинации за прошедший объем трафика.

Not calculated minutes (term)Количество минут, не тарифицированных терминирующим оператором.

Not calculated minutes (orig)Количество минут, не тарифицированных оригинирующим оператором.




При просмотре детального списка звонков отображаемые поля с параметрами звонков можно регулировать в секции “CDR parameters”, которая вызывается кнопкой “Show CDR parameters”.

ПолеНазначение
IpАдрес шлюза оператора.

OperatorНазвание оператора, под которым он заведен в AMBS.

PortПорт MVTS-сервера.

UserПользователь, зарегистрированный на MVTS-сервере.

Bytes inКоличество байт, принятых от оператора.

Bytes outКоличество байт, отправленных оператору.

Number in(Source)Входящий номер, с которого был отправлен звонок на входе.

Number out(Source)Входящий номер, с которого был отправлен звонок на выходе.

Number bill(Source)Входящий номер, с которого был отправлен звонок.

Number in(Destination)Номер, пришедший от оператора.

Number out(Destination)Номер, отправленный оператору.

Number bill(Destination)Тарифицируемый номер.

CostСтоимость звонка.

PriceЦена единицы времени звонка.

Connection timeВремя начала звонка.

Disconnection timeВремя окончания звонка.

Session timeПродолжительность сессии звонка.

Setup timeВремя установки соединения.

Bill dateДата прохождения звонка.

Bill timeВремя прохождения звонка.

Local timeЛокальное время просматриваемого оператора. Если будет задан параметр “Time difference”, время в этом поле покажет время с учетом заданной разницы.

HostХост-сервер, через который пошли звонки.

ConfidИдентификатор конфигурации, которая использовалась для звонков.

CallidИдентификатор звонка.

Disconnect codeКод MVTS-сервера при завершении звонка, показывает сторону, инициировавшую разрыв.

Q931 codeКод, генерируемый MVTS-сервером при завершении звонка. Детальнее коды описаны в “Приложении А” данного руководства.

CodecКодек, использованный при прохождении звонка (входящий и исходящий).

Proxy modeИспользовался режим проксирования, или нет.

Route retriesКоличество попыток маршрутизации звонка.

QOSПараметр качества сервиса. Берется с MVTS-сервера.

Lar fault reasonПоле MVTS-сервера, коды причин отказа.

Gatekeeper idПоказать идентификатор гейткиперов для звонков.


Необходимо учитывать, что вывод списка звонков при их числе более 1000 может существенно замедлить работу браузера. Рекомендуется вначале определить количество звонков, просмотрев суммарную статистику, а затем переходить к просмотру детальной статистики по звонкам.
