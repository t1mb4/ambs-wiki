Operator Route Parameters. Настройка маршрутизации.

В этой форме можно задать вручную настройку маршрутизации для вызовов любого оператора.
Для каждого направления операторов можно установить следующие параметры:

  • Первоочередность направлений. Список из 3 первых ID-маршрутов, куда будет уходить трафик в первую очередь;
  • Прямого направления (direct route). Безусловная маршрутизация трафика без контроля цены и качества;
  • Лист запрещенных направлений. Список ID-маршрутов, куда звонки по этому направлению не будут проходить;
  • Приоритет выбора направлений: по цене, по качеству, по ASR и т. д.;
  • Включить или отключить наличие брекаутов у ID-маршрутов при их выборе;
  • Отключить возможность прохождения трафика по более короткому коду, если такой имеется в тарифе оператора;
  • Закрыть выбранный код у оригинатора. Трафик этого оператора по этому коду приниматься не будет.


Данный раздел позволяет настраивать маршрутизацию как для конкретного оператора, так и для всех доступных операторов
в целом (--All operators--). Маршрутизацию можно задавать на конкретный код (single code) или на группу кодов (destination).



Визуально форма состоит из двух частей. Первая (левая часть) отвечает за выбор сайтов операторов и направлений, для которых будет производиться ручная настройка маршрутизации. Вторая (правая часть) используется для задания параметров маршрутизации выбранным сайтам на указанные направления.

Левая часть формы состоит из следующих полей:

ПолеНазначение
Direction typeСлужит для выбора направлений, для которых будет производиться настройка маршрутизации.
Может принимать значения:
originate – будут производиться настройки для направлений оригинации трафика.
Значение по умолчанию – originate.

Select siteПозволяет выбрать сайт конкретного оператора или всех операторов в целом.
Может принимать значения:
All enabled – Позволяет просмотреть все активные на данный момент настройки по выбранному типу трафика;
All operators – Позволяет настроить маршрутизацию по выбранному коду(ам) всем операторам в целом;
Сайт оператора – Сайт конкретного оператора. Все настройки маршрутизации будут работать только для этого сайта.
Значение по умолчанию – All enabled.

Select countryФильтр по стране, выводит на экран направления только выбранной страны из базы системы.
Примечание: Можно производить поиск требуемого направления с использованием данного фильтра страны или сразу по нужному коду, если он известен заранее.


Select route byФильтр типа отображаемых направлений.
Может принимать значения:
Single code – Будут показаны только единичные коды;
Destinations – Будут показаны как единичные коды, так и группы кодов, удовлетворяющие условию выборки;
Значение по умолчанию – Single code.
Примечание 1: В некоторых ситуациях удобнее использовать группы кодов, например, настроить для всех операторов первым выбором определенного оператора на все коды московских мобильных операторов.

Примечание 2: Следует обратить внимание на то, что «Single code» и «Destinations» с точки зрения системы это две разные функциональности. В случае использования типа «Single code» – система рассматривает набор цифр, составляющий заданный код, в то время как при использовании типа «Destinations» – система работает с конкретными id-номерами записей из базы направлений.

Select rows countКоличество строк направлений, показываемых за один раз на экране.
Может принимать значения: [10...100] или All
Значение по умолчанию – 20.

Filter optionsФильтр отображения направлений по статусу параметра маршрутизации.
Может принимать значения:
Select options – Неактивный фильтр. В таком режиме будут показаны все направления как с активными настройками маршрутизации, так и без них. Обычно используется для поиска направления при настройке маршрутизации.
Enabled only – Показывает только направления с активированными настройками маршрутизации. Обычно используется для быстрого нахождения уже используемых настроек, для их изменения или удаления.
Closed only – Показывает только направления которые были закрыты, со статусом – «Closed».
Значение по умолчанию – Select option.

Filter codeФильтр по коду направления. На дисплее ПК будут показаны все направления из базы системы, которые удовлетворяют условию данного поля.


Правая часть формы состоит из полей:

ПолеНазначение
Direct routeПрямое направление. Позволяет задать один прямой ID-маршрута для выбранного направления(ий). Все другие ID-маршрутов, которые имеются на код выбранного направления будут игнорироваться. Если звонок по этому направлению был отбит ID-маршрута, то звонок больше никуда не пройдет.
Примечание: При использовании этого режима контроль над уровнем цен не используется, т.е. можно сработать в «минус».


Route 1, 2, 3Первоочередные ID-маршрутов, на которые будут направляться звонки по выбранному направлению(ям) впервую очередь. Если звонок был отбит этими ID-маршрутов, далее система будет пытаться отправить звонок по стандартной схеме, с учетом цен и других параметров сортировки.

Deny routeСписок запрещенных ID-маршрутов. Для этого оригинирующего оператора по этому направлению(ям) ID-маршрутов из этого списка являются запрещенными.
Примечание: При добавлении ID-маршрута в «список запрещенных» следует выбрать ID-маршрута из выпадающего списка «deny route» и нажать кнопку «set_route». Чтобы убрать ID-маршрута из «списка запрещенных» следует в «запрещенных маршрутах» убрать галочку с этого ID-маршрута и нажать кнопку «unset_route».


Without breakoutsПозволяет использовать более длинные коды других ID-маршрутов (если такие имеются) вперед первоочередных ID-маршрутов. Если функция включена, и заданы конкретные ID-маршрутов в полях (route 1,2,3), тогда перед направлением звонка на эти ID-маршрутов система будет пытаться отправить звонок на другие ID-маршрутов, предоставляющих более длинные коды по данному направлению.

Пример:

У нас есть несколько ID-маршрутов, предоставляющих следующие направления:
ID-маршрута Б – направление с кодом «7»
ID-маршрута В – направление с кодом «7495»
ID-маршрута Г – направление с кодом «7499»

От оригинатора поступил звонок с номером – 7495.....
Для этого оператора установлены «route 1» -> «ID-маршрута Б» и включена опция «Without breakouts».

При такой схеме система сначала направит звонок на «ID-маршрута В» и только потом на «ID-маршрута Б».

Route params enableАктивирует настройки. Это поле следует активировать, если необходимо использовать заданные пользователем настройки маршрутизации.
Если поле неактивно, настройки будут сохранены, но использоваться не будут.

Примечание:
  • Если настройки маршрутизации активны и используются, в листе направлений строка с кодом будет подсвечена зеленым цветом;
  • Если настройки присутствуют, но строка имеет стандартный серый цвет – настройки маршрутизации «не активны»;
  • Для настроек активации следует выбрать это направление из списка и в поле «Route params enable» для него поставить галочку.


Sort typeТип сортировки списка терминационных направлений.
Может принимать следующие значения:
Price – Список доступных маршрутов, будет сортироваться по минимальной цене;
ASR – Сортировка по лучшему показателю ASR;
ACD – Сортировка по лучшему показателю ACD;
Qi – Сортировка по общему индексу качества.
Значение по умолчанию – Price.

CloseЗакрыть направление. Вызовы выбранного оператора по закрытому коду приниматься не будут.

Proper rejectЗапрещает прохождение звонков на более короткие коды.
Например: Если данная функция настроена для кода 79, звонки настраиваемого оператора будут проходить на ID-маршрутов, дающих только код 79 и более длинные. Все ID-маршрутов, которые дают код 7 будут игнорироваться.


В форме присутствуют следующие кнопки управления:

КнопкаНазначение
Удаляет настройки параметров маршрутизации у кодов (групп направлений), выделенных в списке направлений.

Вызывает форму для добавления комментариев.

Сохраняет внесенные изменения в настройки.

Выводит на экран лист кодов (групп кодов) с учетом заданных в форме параметров отображения.

Необходимо учитывать следующие моменты при работе с данной формой:
  1. Можно производить поиск требуемого направления с использованием фильтра страны или сразу по нужному коду, если он известен заранее;
  2. Для сокращения списка показываемых кодов при поиске, правильнее указывать фильтр типа направлений, или пользователь будет настраивать маршрутизацию на конкретный код (single code) или на группу кодов (destinations);
  3. При настройке «Direct route», маршрутизация будет идти напрямую без контроля за ценой и статусом направления у ID-маршрута. Т.е., если цена терминации выше цены оригинации или этот ID-маршрута закрыт, звонок все равно будет маршрутизироваться на выбранный ID-маршрута;
  4. Если настройки маршрутизации активны и используются, в листе направлений строка с кодом будет подсвечена зеленым цветом. Если настройки присутствуют, но строка имеет стандартный серый цвет – настройки маршрутизации «не активны»;
  5. Коды для настроек маршрутизации берутся из общей базы кодов системы и не привязаны к тарифам операторов. Данная схема позволяет настроить маршрутизацию более гибко и точно.
