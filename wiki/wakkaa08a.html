Edit operators. Просмотр личных параметров


Данный раздел системы является основным для работы с операторами. Здесь дилер может просмотреть всю необходимую информацию по доступным ему операторам: реквизиты и контактные данные, тарифные планы, закрепленные за оператором, шлюзы операторов (gateways), привратники (gatekeepers) и пользователей (users).
На следующем рисунке показан исходный вид формы «Edit operators»:



Доступного оператора следует выбирать из выпадающего списка «Select operators». Параметр «All operators» позволяет просматривать закрытых операторов (по умолчанию, после того как статус оператора изменяется на «закрыт», он не отображается в выпадающем списке).
После того как будет выбран оператор, система загрузит информацию по всем настройкам этого оператора. Внешний вид формы оператора будет выглядеть следующим образом, Cм. рис.:



Форма оператора разделена на три секции с параметрами выбранного оператора:
  • Секция параметров оператора и его контактной информации;
  • Секция тарифных планов оператора;
  • Секция шлюзов, привратников и пользователей оператора.

1. Описание секции параметров оператора и его контактной информации:




ПолеНазначение
Operator descriptionОписание оператора. Сокращение или аббревиатура названия компании оператора.

Organization nameПолное название компании.

AddressАдрес компании оператора.

PhonesКонтактные телефоны компании.

FaxФакс компании оператора.

Agent orig/term (%)Процент с оригинации/терминации, который будет получать агент, работающий с этим оператором.

ManagerНазначенный данному оператору менеджер, который ведет этого оператора в процессе работы.

Fin. contact personКонтакт для финансовых вопросов.

Tech. contact personКонтакт для технических вопросов.

CompanyКомпания, от лица которой работаете с этим оператором.

Invoice parametersПараметры напоминания о выставлении счета, бывают нескольких типов:
Auto invoice – Автоматическое напоминание. Отправляет напоминание на электронный адрес менеджера, курирующего этого оператора, о сроке выставления счета оператору;
Mail invoice – Создается и отсылается готовый инвойс оператора на электронный адрес менеджера;
Mail detalization – Создается и отсылается CDR-файл с детализацией звонков за прошедший период на электронный адрес менеджера.

Invoice periodПериод, как часто система будет отправлять менеджеру напоминания, счета или CDR-ы данного оператора:
Every day – Каждый день;
Every week – Каждую неделю;
Every month – Каждый месяц;
2 times per month – Два раза в месяц.

Max creditЕсли данный параметр имеет ненулевое значение – система будет работать с данным оператором в режиме «Кредит» до окончания суммы его кредита (указанного в данном поле). Как только весь кредит будет исчерпан, оператор не сможет никуда позвонить (но на этого оператора звонки будут проходить).
Если значение поля ноль, система будет работать с оператором в режиме «Предоплаты», т.е. без ограничений.

Time differenceРазница между часовыми поясами операторов. По умолчанию имеет значение – 0. Если выставлена любая другая величина, то эта разница будет автоматически использоваться при создании инвойсов.

Sort routes by…Определяет, по какому фактору выбирать список доступных направлений для этого оригинирующего оператора: по минимальной цене, по ASR, по ACD или по индексу качества Qi.

Minimal percentПоле, отвечающее за прибыль от этого оператора. Может иметь отрицательное значение.

Operator statusТекущий статус оператора:
Working – Оператор работает;
Closed – Оператор закрыт.

Q931 codeТип кода, который будет отсылаться этому оператору, если его звонки не будет отбивать сама система. Например, если оператор работает по кредитной системе и у него закончился кредит.

Operator typeТип оператора, определяет его статус в системе и уровень его доступа. Существует несколько типов операторов:
Partner – Оператор, с которым работают непосредственно;
Retail – Клиент оператора типа «Partners», имеющий свои тарифные планы в системе;
Virtual – Виртуальный клиент для оператора типа «Retail». На данный момент не используется в системе.

2. Описание секции сайтов тарифных планов оператора:


Сайт – это список шлюзов и назначенных им тарифных планов. Вся работа оператора строится «от» его сайтов. Каждый оператор изначально имеет по одному сайту, при необходимости количество сайтов может быть увеличено.



Система разделенных сайтов для каждого оператора была сделана для увеличения гибкости параметров соединения. Такая схема позволяет легко организовать работу оператора с отдельными тарифными планами для разных ip-адресов: например, оператор предоставляет два ip-адреса, по одному из которых он предоставляет направления среднего качества, а по второму, те же самые направления, но более лучшего качества.

При создании нового оператора для него автоматически будет создан один пустой сайт с его именем. Каждый сайт позволяет закрепить за оператором по одному тарифному плану следующих типов:

ТипОписание
Terminate rateПросмотр закрепленного выбранного тарифного плана на терминацию.

Originate rateПросмотр закрепленного выбранного тарифного плана на оригинацию.

Saler rouming rateПозволяет выставить роуминговый терминационный тариф.

Buyer rouming rateПозволяет выставить роуминговый оригинационный тариф.

Operator clients rateПросмотр клиентского тарифа, который данный оператор сможет в последствии регулировать сам для своих клиентов. Часто используется для переговорных пунктов.

Internet terminate rateСекция для закрепления терминационных тарифных планов для интернет операторов. Используется для работы модуля «Net Flow».

Internet originate rateСекция для закрепления оригинационных тарифных планов для интернет операторов. Используется для работы модуля «Net Flow».

hunt modeРежим распределения нагрузки.
Может принимать следующие значения:
no resort(1) – Функциональность отключена, и трафик направляется последовательно на все шлюзы оператора;
first to end(2) – Адреса циклически меняются (первый адрес становится последним, второй – первым, и т. д.), смена шлюза происходит по завершении звонка;
by calls count(3) – Адреса меняются в зависимости от загруженности шлюза (т. е. трафик направляется на наименее загруженный в данный момент времени адрес);
by calls count/capacity(4) – Адреса меняются в зависимости от загруженности шлюза с учетом параметра «capacity».
Значение по умолчанию – no resort.

Примечание:
Кроме созданных вручную тарифных планов (в форме «Edit tariffs»), к сайту оператора можно прикрепить оригинационный план «Себестоимость», это позволит оператору звонить по всем доступным направлениям по себестоимости этих направлений. При создании сайта, за ним не закреплены никакие тарифные планы. Без тарифного плана через этот сайт звонки проходить не будут.

3. Описание секции шлюзов, привратников и пользователей оператора:


Ниже секции сайтов оператора расположена секция для шлюзов, закрепленных за выбранным сайтом оператора.
В заголовке секции присутствует панель навигации, позволяющая просматривать один тип шлюзов за раз:



ПолеНазначение
GatewaysШлюз. Аппаратно-программный комплекс, обеспечивающий обмен данными между сетями разных типов.

GatekeepersПривратник. Выполняет функции управления зоной сети H.323.

Default usersАбонент сети IP-телефонии (шлюз, терминал), регистрирующийся на привратнике по RAS-протоколу, с использованием логина и пароля.

Configured usersАбонент сети IP-телефонии (шлюз, терминал), регистрирующийся на привратнике по RAS-протоколу, с использованием логина и пароля и имеющий свой ip-адрес и другие параметры конфигурации.


Далее в панели секции расположен фильтр «active ip», позволяющий просматривать необходимую информацию, не отображая всего остального.
Фильтр может принимать следующие значения:

ЗначениеОписание
Active ipПоказывает все активные на данный момент времени шлюзы на всех рабочих софтсвичах.

AllПоказывает все шлюзы закрепленные за этим сайтом, активные и закрытые.

All on working serversПоказывает все шлюзы (активные и закрытые), заведенные только на активных в данный момент времени софтсвичах.

All on closed serversПоказывает все шлюзы (активные и закрытые), заведенные только на выключенных в данный момент времени софтсвичах.


Секция шлюзов оператора (GW – Gateways)


По умолчанию всегда открывается форма шлюзов оператора (gateways).


Описание полей формы:

ПолеНазначение
IP addressIP-адрес шлюза. Поле обязательно для заполнения. В систему заносятся только уникальные адреса шлюзов.

Примечание: Записи адресов типа:
111.222.333.1–255
111.222.333.44/29
работать не будут. Все подсети записываются через маску адреса.


Gateway IDИдентификатор (название шлюза), уникален для каждого шлюза.

Proxy typeПараметр проксирования трафика:
disabled – Проксирование только сигнального трафика;
enabled – Проксирование сигнального и медиа (речевого, мультимедийного) трафика.

Gateway modeРежим работы шлюза:
0 (nothing) – Шлюз отключен;
1 (originate) – Оригинация. Шлюз используется только для отправки трафика на операторов;
2 (terminate) – Терминация. Шлюз используется только для приема трафика от операторов;
3 (both) – Все. Трафик отправляется, и принимается с этого шлюза.

Dst_translateСодержит правила преобразования номера вызываемого (dst_translate) абонента. (Необязательное поле).
Как правило, состоит из двух частей, разделенных символом «/»
  • выражение regexp, задающее строку поиска
  • строка замены

In_dst_translateПоле трансляции входящего номера с задаваемого адреса оператора. (Необязательное поле).
Указывается параметр трансляции, который должен преобразовывать номер на входе.
Используется только для входящих звонков. Например, префикс, который должен отрезаться при получении звонков с этого адреса, этого оператора.

IP serverMVTS-сервер, на котором заведен данный шлюз.

StateСтатус данного шлюза:
Closed – отключен;
Opened – работает.


Секция привратников оператора (GK – Gatekeepers)


Для просмотра формы привратников оператора следует выбрать пункт «gatekeepers» в меню шлюзов оператора.
Секция привратников имеет следующий вид:



Внешний вид секции привратников аналогичен виду секции шлюзов оператора.

Описание полей формы:

ПолеНазначение
Operator ipIP-адрес шлюза.

Proxy typeПараметр проксирования трафика:
disabled – проксирование только сигнального трафика;
enabled – проксирование сигнального и медиа (речевого, мультимедийного) трафика.

Gateway modeРежим работы шлюза. Может принимать следующие значения:
0 – Шлюз отключен;
1 – Оригинация. С этого шлюза проходит только оригинация трафика;
2 – Терминация. С этого шлюза проходит только терминация трафика;
3 – Все. Трафик отправляется и принимается с этого шлюза.
По умолчанию – 3.

Dst_translateСодержит правила преобразования номера вызываемого абонента. Как правило, состоит из двух частей, разделенных символом «/»
  • выражение regexp, задающее строку поиска
  • строка замены
Необязательное поле.

In_dst_translateПредварительная трансляция, выполняемая при поступлении звонка от этого шлюза, перед вызовом поиска по объектам набора (dial peers).
Параметры полезны, когда какой-либо шлюз присылает номера, несоответствующие формату Е164.

Server IPMVTS-сервер, на котором прописан этот привратник.

Gatekeeper Имя привратника.


Gatekeeper userИмя пользователя для авторизации на привратнике оператора.

Gatekeeper passwordПароль привратника. Используется системой совместно с внутренним именем для тех же целей.

StateСтатус данного привратника:
Closed – отключен;
Opened – работает.
