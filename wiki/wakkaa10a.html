Формат полей CSV-файла с направлениями


Если имеется файл с большим количеством направлений в формате Excel, этот файл можно подготовить и импортировать в тарифный план IP Studio AMBS через функцию импорта направлений.

Для импорта направлений в IP Studio AMBS, нужно привести записи направлений в Excel-е к следующему виду:



Порядок полей в Excel-файле с направлениями должен полностью соответствовать полям направлений в системе.
Рассмотрим значение каждого поля в том порядке, в котором они должны присутствовать в файле:

ПолеНазначение
DESTNAMEНазвание направления.

CITYНазвание города или региона

TYPEТип направления (пропер, сотовые, регион, город и т.д.)

COUNTRY CODEКод страны загружаемого направления.

DEST CODEКод направления внутри страны. Код страны и код направления внутри страны должны быть записаны в отдельных ячейках.

PRICEЦена направления.

FIRST ROUNDМинимальная длительность звонка. Если длина звонка меньше количества секунд в этом поле, длительность звонка будет приравнена к указанному здесь периоду.

ROUNDОкругление длительности разговора. По умолчанию устанавливается 1 секунда, т.е. посекундная тарификация.

START DATEДата начала работы направления. Тарифное направление считается рабочим с первой секунды этой даты.

STOP DATEДата окончания работы направления. Тарифное направление считается нерабочим с первой секунды этой даты. После этой даты звонки на это направление по этой цене проходить не будут.

GRP (Group)Группа направлений, к которой будет принадлежать данное направление. Используется для настройки маршрутизации на основе групп направлений операторов.

DENY_GRP
(Deny group)		Запрещенная группа направлений, куда (откуда) звонки по этому направлению от (на) данного оператора проходить не будут. Используется для настройки маршрутизации на основе групп направлений операторов.

MINSECМинимальная продолжительность звонка. При длительности звонка меньше указанного в этом поле количества секунд, звонок будет считаться бесплатным.

UNPAID
(Unpaid sec)		Неоплачиваемые секунды. Количество бесплатных секунд с начала разговора.

DISCRДискретность. Количество секунд, составляющее единицу цены. По умолчанию установлено значение – 60 секунд, т.е. единица цены (указанная в поле “Price”) будет сниматься за каждые 60 секунд разговора.

STATE (Work/Close)Закрыть направление. Позволяет временно закрыть данное направление. Если в этом поле стоит галочка, то звонки на это направление проходить не будут.

TZ_ID
(Time zone)		Период времени, по которому работает направление.


После того, как все направления в файле будут преобразованы к виду, показанному на рисунке, файл нужно сохранить в формате – *.CSV (MS-DOS) разделитель точка с запятой или *.CSV (разделители – запятые) (Система поддерживает оба формата).

В форме “Edit tariffs” (Tariffs – Edit tariffs) создать или выбрать тарифный план, куда будет производиться загрузка направления из файла.
С помощью кнопки “Browse” (Обзор) в форме, выбрать подготовленный CSV-файл с направлениями и нажать кнопку .

Показанных полей достаточно, чтобы биллинг добавил в тарифный план направления из CSV-файла. Все остальные, отсутствующие в записи направлений поля, после загрузки будут иметь стандартные для системы значения, показанные на первом рисунке этого раздела.
