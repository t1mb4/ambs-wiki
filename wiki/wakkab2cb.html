Настройка карточной платформы в системе IP Studio AMBS


Непосредственно в самой системе, весь процесс настройки состоит из следующих этапов:
1. Заведение тарифного плана для карточного лота;
2. Создание карточного лота;
3. Заведение оператора карточной платформы.

Рассмотрим подробнее каждый из пунктов:
  • в первом пункте следует задать стандартный (для системы AMBS) тарифный план, на базе которого впоследствии будет рассчитываться баланс карточек. Более детально, как добавлять тарифный план следует прочитать в руководстве пользователя по системе IP Studio AMBS в главе по заведению и работе с тарифными планами;

  • вторым пунктом следует создать новый карточный лот, пользователь может сделать это, воспользовавшись разделом «Lots&Cards»;

  • третьим пунктом следует завести оператора, это необходимо делать в случае, когда необходимо использовать карточный модуль в связке с основной системой AMBS. Также здесь важно учитывать, что в тех случаях, когда пользователь планирует пропускать несколько типов трафика через один сервер доступа, они должны различаться префиксами, что бы в последствии их можно было различать и анализировать.
В этом случае, при создании оператора карточной платформы, если сервер доступа будет слать программному коммутатору трафик с добавкой префикса необходимо указать этот префикс у этого оператора, что бы AMBS отрезала этот префикс на входе. Сам процесс добавления оператора ничем не отличается от процесса добавления обычного транзитного оператора в базу системы. Более детально можно прочитать об этом в руководстве пользователя по системе IP Studio AMBS в соответствующей главе по заведению и работе с операторами.
