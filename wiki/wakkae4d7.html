Форма “Edit Tariffs”


Посмотреть пример добавления тарифного плана



У каждого оператора зведенного в системе должен быть свой тарифный план на оригинации или терминацию трафика в зависимости от того, что он предлагает.
Данная форма предназначена для организации и работы с тарифными планами операторов.


Визуально форма разделена на три части:

  • левая часть для выбора тарифных планов и импорта направлений из файлов;
  • средняя часть содержит набор фильтров, для организации выборки в тарифных планах;
  • правая часть используется для добавления одиночных направлений в тарифный план.


Поля формы:


ПолеНазначение
Select rateСписок с уже существующими в системе тарифными планами.
После выбора одного из планов, будет показан его лист направлений.

Rate typesТип тарифных планов (далее просто тариф). Необязательное поле для сортировки.
Позволяет классифицировать все тарифные планы по их назначению.
Может принимать следующие значения:
Origination/Termination – Все транзитные тарифные планы (планы у которых стоит тип – orig или term);
Roaming – Роуминговые тарифныепланы;
Originate – Только оригинационные тарифные планы;
Terminate – Только терминационные тарифные планы;
Cards – Тарифные планы карточной платформы;
Clients – Клиентские тарифные планы;
Internet – Тарифные планы интернета.
Значение по умолчанию: не задано.

Rate nameСтрока для ввода имени нового тарифного плана или для изменения имени выбранного тарифа.

Select currencyСписок используемых в системе валют. Служит для выбора валюты, в которой будет работать тарифный план, будут вестись цены.
Может принимать значения любой из валют заведенных в системе.
Значение по умолчанию – валюта системы.

Rename destinationsПараметр загрузки тарифных направлений из файла.
Переименовывать направления при их загрузке из файла.
Система будет загружать коды с учетом их новых названий в файле, если в системе уже присутствовали эти коды, их названия будут заменены на новые.

Open earlier closed codesПараметр загрузки тарифных направлений из файла.
Удаляет в форме “Route Parameters” все правила на закрытие кодов по загружаемым кодам.
Будут удалены все правила заданные на терминацию, в режиме “single code”.

Rate stateПараметр загрузки тарифных направлений из файла.
Определяет изменение статуса загружаемых направлений в тарифный план, если эти направления уже присутствуют в нем.
Может принимать следующие значения:
As in file – изменить статус загружаемых направлений как указано в файле (если не указано – статус «Открыто»);
All open – изменить статус загружаемых направлений на «Открыто»;
Inherite state – наследовать статус направления из тарифного плана (если направление было закрыто, новое также будет закрыто);
All close – изменить статус загружаемых направлений на «Закрыто»;
Значение по умолчанию: Inherite state

Select countryФильтр по стране. Позволяет просмотреть в тарифе все направления выбранной страны. Поле работает сразу с двумя частями формы: фильтрует направления тарифного плана и выводит все направления этой страны в правую часть самой формы для добавления одного из них в тарифный план (при необходимости).

Select typeФильтр по типу направлений (пропер, сотовые, городские, региональные и т.д.).
Поле работает с двумя частями формы: фильтрует направления уже созданного тарифного плана и выводит соответствующие направления в правую часть самой формы.

Full codesТип отображения кодов в направлениях в листе тарифного плана. Если галочка стоит, все направления будут показаны с полным списком кодов входящих в состав каждой записи. Если галочка убрана, все записи будут показаны в сокращенном виде – будут показаны только первые три кода направлений в каждой записи.

Destination numberФильтр по первым цифрам кода направления. В листе направлений тарифного плана будут показаны только те направления, первые цифры которых соответствуют заданным в этом поле.
Знак "%" используется для расширенного поиска, например, чтобы найти все мобильные коды России заведенные в тарифном плане, можно указать шаблон поиска как «79%".

Filter close date (yyyy-mm-dd)Фильтр по дате окончания работы направлений в тарифном плане.
В тарифном плане будут показаны только те направления, у которых в поле “Stop date” дата совпадает с указанным значением.
Указываемая дата должны иметь формат: ГГГГ-ММ-ДД.

Filter start date (yyyy-mm-dd)Фильтр по дате начала работы направлений в тарифном плане.
В тарифном плане будут показаны только те направления, у которых в поле “Start date” дата совпадает с указанным значением.
Указываемая дата должны иметь формат: ГГГГ-ММ-ДД.

Status filterФильтр видимости направлений тарифного плана.
Может принимать одно из двух значений:
Current – Показывать только активные в данные момент направления.
Под активными направлениями понимаются все направления, у которых текущая дата попадает в период между “Start date” и “Stop date” этих направлений.
History – Полный список направлений. Будут показаны все направления, включая записи, которые работали в прошлом, и будут работать в будущем.
Значение по умолчанию – current

Rates status filterФильтр статуса направлений. Будут показаны только направления с выбранным статусом.
Может принимать следующие значения:
All rates – Полный список направлений
Closed only – Только закрытые направления
Opened only – Только открытые направления
Значение по умолчанию – All rates.

Time zones filterФильтр по временным периодам. В выбранном тарифном плане будут показаны только те направления, которые активны в выбранный временной период.
Может принимать значение любого периода времени, заведенного в системе.
Значение по умолчанию – Any Time Zones (все периоды).

Select deny groupФильтр по запрещенной группе. Показывает все направления в выбранном тарифном плане за которыми закреплена выбранная запрещенная тарифная группа.

Excel export filterСписок с вариантами отображения кодов в направлениях тарифного плана при его экспорте в CSV-файл.
Может принимать следующие значения:
Format_string – Система будет экспортировать все записи направлений с учетом регулярных выражений системы (коды направлений будут записаны в универсальной форме с сокращениями).
Comma_delimited – Все коды направлений будут записаны через запятую.
By_codes – Все коды будут записаны отдельно. Если одно направление состоит из группы кодов, такая строка будет разделена на несколько строк, по одному коду в каждой.
Значение по умолчанию – Format_string.

Destname filter (правая часть формы)

Производит поиск направления по его названию(или части названия).
Для поиска по части названия используются системынве символы – "%".
Примеры:

  • Найти все направления с названием “Moscow”: %Moscow%
  • Найти все направления начинающиеся с “USA”: USA%
  • Найти все направления с названием заканчивающимся на “York”: %York



Также в форме присутствуют следующие кнопки управления:


КнопкаНазначение
BrowseПозволяет выбрать CSV-файл с направлениями для его загрузки в выбранный тарифный план или для его конвертации.

Запускает процесс конвертирования выбранного CSV-файла с направлениями в формат системы.
После окончания процесса полученный файл можно сохранить на локальный ПК со страницы “Waiting form”, которая появится по завершении процесса.
Детально данная функциональность описана ниже, в разделе “Конвертирование прайс-листа в формат системы”.

Загружает выбранный CSV-файл с направлениями в тарифный план.
По окончании процесса будет дана ссылка(«Show load rate LOG») на лог загрузки.


Примечание: При загрузке уведомлений об изменение цен, каждое уведомление (изменение цен указанной датой)
должно загружать отдельным файлом. Отдельно файл изменений от “02–02–2009” и отдельно от “08–02–2009”.

Отменяет последнюю загрузку направлений из файла.

Создает новый пустой тарифный план с именем, указанным в поле “Rate name”.
Примечание: У каждого оператора должен быть свой отдельный тарифный план.

Обновляет информацию об имени и валюте выбранного тарифного плана, если были внесены изменения.

Позволяет добавить комментарий по оператору.

Удаляет выбранный тарифный план. Можно удалить только когда тарифный план не закреплен за сайтами операторов.

Выводит на экран лист направлений выбранного тарифного плана с учетом заданных параметров фильтров.

Filter (правая часть экрана)Фильтр по существующим в системе направлениям. Используется для поиска и добавления единичного направления в тарифный план.

Add destination (в правой части экрана, показывается после поиска направления)Добавляет выбранное в правой части формы единичное направление в тарифный план. по умолчанию, после добавления, это направление будет иметь нулевую цену и будет закрыто. Его нужно активировать и внести необходимые изменения.


После выбора тарифного плана из списка, будет показан лист с его направлениями с учетом всех установленных фильтров:



В данном случае был использован фильтр активных направлений – current, этот фильтр используется по умолчанию, чтобы не загружать экран лишней информацией – направлениями, которые в данный момент не активны.


Если изменить значение фильтра активности на history, система покажет полный список направлений в тарифном плане: направления, которые работали раньше, работают сейчас и будут работать в будущем. Например, предположим, что сегодня – 12–02–2007, тогда при просмотре выше показанного тарифного плана с использованием фильтра активности “history”, мы получим следующий вид:



Как видно из рисунка, в данном случае направлений больше. Обратите внимание на даты активности направлений. Разберем их по порядку:


'Kyrgyzstan Bishkek (996312)' – Направление закрыто галочкой Closed и имеет цену 0 – так добавляется в тарифный план единичное направление. После добавления такое направление нужно отредактировать с учетом его цены и периода активности. На данный момент на это направление трафик проходить не будет.

'Russia Moscow (7495)' – Для этого направления присутствуют две записи: первая с периодом работы “2006–02–10”-»2007–06–10” будет активна на данный момент времени. Текущая дата 2007–02–12 попадает в интервал между Start и Stop датами работы направления. Вторая запись “2007–06–10”-»9999–01–01” с новой ценой будет активна в будущем, с 2007–06–10.

'Russia St.Petersburg(7812)' – Также имеем две записи: первая с периодом работы “2005–01–01”-»2006–02–10” – прошедший период работы, по которому направление работало раньше со старой ценой. Вторая запись “2007–02–10”-»9999–01–01” активна на данный момент времени – запись с текущей ценой.

Примечание: Дата вида “9999–01–01” означает в системе открытую дату, без конкретного срока завершения работы.

При необходимости выбранный тарифный план можно сохранить в CSV-файл, воспользовавшись кнопкой “Excel version” внизу листа направлений. При этом все направления в файле будут записаны с учетом выбранного вами формата записи кодов: format_string, comma_delimited или by codes.
