Форма “Surcharge”


Форма дополнительных платежей позволяет создавать схемы, которые можно закреплять за картами лотов и при прохождении звонков, с этих карт будут сниматься дополнительные платежи по этим схемам. Вид формы представлен на следующем рисунке:



Функциональность этой формы практически не отличается от функциональности работы с формой тарифов.

Описание полей формы:

ПолеНазначение
Select or new surcharge nameПозволяет выбрать из выпадающего списка уже созданные планы для просмотра или внесения изменений. Чтобы добавить новый план следует вписать его название в пустое поле рядом. Значение по умолчанию: Select surcharge plan.

Select countryВыпадающий список со странами, позволяет выбрать номера ANI или DNIS конкретной страны для добавления их в план дополнительных платежей.
Значение по умолчанию: Select country.

Select typeПозволяет просмотреть номера ANI или DNIS только конкретного типа: region, cell, city и т.д.

Rates list rows countКоличество записей плана, которые будут показаны одновременно на дисплее ПК. Может принимать значения: 10,20,30,50,100 или All (все).
Значение по умолчанию: 20.

Dest numberФильтр, позволяющий просмотреть только те записи у которых первые цифры ANI или DNIS совпадают с указанным в этом поле значением.

Current / HistoryТип просматриваемых записей в плане, только активные на данный момент времени или все записи, соответственно.

Format string / Comma delimitedТип записи кодов, при экспорте плана дополнительных платежей в файл формата .csv. С помощью регулярных выражений AMBS или последовательно, через запятую, соответственно.


В форме расположены следующие кнопки управления:

КнопкаНазначение
FilterЗапускает процесс вывода на экран плана дополнительных платежей с учетом условий, заданных пользователем.

Add surchargeДобавляет новый план платежей с именем, указанным в поле рядом с выпадающим списком «Select surcharge plan».

Delete surchargeУдаляет выбранный план дополнительных платежей.

Update surchargeСохраняет изменения, внесенные в название плана платежей.


Система поддерживает два условия добавления дополнительных платежей:

1. Платежи, закрепленные за DNIS, за определенным номером доступа к карточной платформе, т.е. когда абонент будет звонить через этот номер доступа и при этом будут выполняться другие условия снятия платежа, с этого абонента будет сниматься дополнительный платеж;
2. Платежи, закрепленные за определенным номером, с которого звонит абонент (ANI). В тех случаях, когда абонент будет звонить с этого номера и будут выполняться остальные условия, с него будет снят дополнительный платеж.

Существует два типа платежей:
  • periodic (периодичные) – Периодические платежи будут сниматься постоянно в течение всей сессии звонка с определенной периодичностью;
  • bong (разовые) – Разовые платежи снимаются один раз за всю сессию разговора после истечения определенного периода времени с начала разговора.

Внешне план дополнительных платежей выглядит следующим образом:



Записи плана платежей состоят из следующих полей:

ПолеНазначение
{empty}Позволяет активировать конкретную запись плана для редактирования или удаления.

AreaРегион, к которому принадлежат ANI или DNIS номера этой записи (не имеет особого значения).

AmountСумма дополнительного платежа.

Surcharge typeТип дополнительного платежа: периодический или разовый.

Area typeУсловие снятия платежа, по ANI или по DNIS.

Period / DelayПериод, с которым будет сниматься дополнительный платеж.
Может принимать значения: bong – разовый, periodic – периодично.
Значение должно быть больше нуля (0 – выключено).

Start dateДата начало работы данной записи дополнительного платежа.

Stop dateДата окончания работы данной записи дополнительного платежа.

CloseПозволяет временно прекратить работу выбранной записи плана дополнительных платежей без удаления записи и изменения ее условий.

Code stringСтрока с номерами ANI или DNIS.


Под таблицей с записями дополнительных платежей находится несколько контрольных кнопок для работы с записями:

КнопкаНазначение
Change amountЕсли пользователю необходимо изменить цену для одной или нескольких записей, при этом оставить старую для статистики – следует воспользоваться этой кнопкой. Пользователю следует изменить стоимость платежа и дату начало работы этого нового платежа. После этого, нажав на эту кнопку в старой записи дата окончания будет изменена на дату начала новой записи и будет добавлена еще одна запись идентичная старой с внесенными изменениями.

Change amount without historyИзменяет данные в записи. Используется, например, когда пользователь случайно внес ошибку и ее необходимо исправить.

Delete amountУдаляет активные записи из плана платежей.

Check allВыделяет все записи в плане дополнительных платежей.

Uncheck allСнимает выделение со всех записей с платежами.

Close allЗакрывает все записи в плане.


Пример создания плана дополнительных платежей.

Необходимо создать план доплат на каждую минуту по направлениям 7495 и 7388, если звонят с определенного DNIS-а.
Для этого нужно выполнить следующие шаги:

1. В пустое поле справа от «Select surcharge plan» ввести название нового плана – Test;
2. Нажать кнопку «Add surcharge» для подтверждения;
3. Добавить направление 7495 в созданный план, со следующими параметрами: surcharge type – periodic; area type – ANI, Period/delay – 60
(снимать дополнительную оплату периодически, если звонят с указанного номера ANI каждые 60 секунд);
4. Добавить направление 7388 в созданный план, со следующими параметрами: surcharge type – periodic; area type – DNIS, Period/delay – 60
(снимать дополнительную оплату периодически, если звонок пришел с указанного номера DNIS каждые 60 секунд);
5. В обоих случаях (3 и 4) для сохранения вводимых параметров следует пользоваться кнопкой «Change amount without history»;
6. Закрепить созданную схему «Test» за определённым лотом.
