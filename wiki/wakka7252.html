Архивирование старых данных в IP Studio AMBS



Начиная с версии АМБС 4.0 (build 2.0) в систему добавлен скрипт позволяющий производить резервное копирование и отчистку старых данных по звонкам из MySql-базы.

Скрипт расположен в директории «billing/bin», по умолчанию:
/usr/local/billing/bin/arch_cdr.pl

Параметры запуска:
КлючОписание
--dateДата в формате 'YYYY-MM-DD', все данные о звонках до этой даты будут включены в резервную копию. Ограничение по дате составляет четыре месяца от текущей даты.
--deleteДанные вошедшие в резервную копию будут удалены из базы.
--pathЗадать путь, куда сохранять файлы резервной копии, по умолчанию путь '$PREFIX/backup/' из файла bill_util.cfg (т.е. /usr/local/billing/backup/)
Путь должен заканчиваться знаком «/».
--helpВыводит на экран справку по параметрам запуска.

Пример использования:
./arch_cdr.pl --date=2008-01-01 --delete --path=/backup/

Примечание: Рекомендуется запускать скрипт из под пользователя 'ipstudio'.
Если резервирование будет производиться в несуществующую директорию из под пользователя 'root', эта директория будет создана с правами 'root'.
В этом случае MySql работающий из под пользователя 'ipstudio' не сможет создать в этой директории файлы резервных копий данных.
