Проверка параметров оператора


Проверку можно провести по следующим этапам:

1. В форме “Edit operators” проверить следующие параметры оператора:
  • Статус оператора – “working”;
  • Наличие активного баланса (если задано значение кредита отличное от нуля);
  • Сайты оператора. Проверить закрепленные тарифные планы (оригинация/терминация);
  • Шлюзы оператора. Проверить:
    • наличие хотя бы одного рабочего шлюза;
    • порт шлюза (1720/5060);
    • тип работы шлюза (оригинация/терминация);
    • адрес меры, на которой заведен шлюз;
    • статус шлюза, должен быть – opened;
Замечание: Подсети (пример: 192.168.0.0/28) можно заводить только для оригинации через маску. Для терминации Мере необходимо указать точный ip-адрес.
Подсети начинаются с четных адресов.

2. Проверить тарифные планы оператора. Проверить тарифные направления, что они присутствуют, имеют не нулевую цену и открыты для работы.

3. В форме “Route test” проверить как система будет маршрутизировать звонки оператора:

Если проверяемый оператор – терминирует трафик:
Выберите любой код предоставляемый оператором. Далее в форме “Route test” выбирите любого оригинирующего оператора с более высокой ценой по этому коду. В поле для номера укажите любой тестовый номер по выбранному направлению. В результате система должна выдать лист доступных для маршрутизации направлений терминации. Нужно проверить, что шлюзы проверяемого оператора присутствуют в этом списке.

Если проверяемый оператор – оригинирует трафик:
В форме “Route test” нужно выбрать проверяемого оператора в выпадающем списке оригинаторов, указать тестовый номер по любому направлению из тарифного плана оператора в поле для номера. Убедиться, что в результате для этого номера система выдаст шлюзы терминации.

Если в тарифных планах оператора есть ошибки, например направления были добавлены не правильно, закрыты или указаны неправильные цены, тогда вместо списка с результатами “Route test” выдаст сообщение, что данное направление отсутствует в тарифном плане оператора или для него нет маршрутов.
