Примеры заведения «Тарифных групп»


Пример 1:


Имеем работающую карточную платформу, на которую дозваниваются из нескольких городов России: Москва, Санкт-Петербург и Ижевск.
Нужно, чтобы один карточный лот обрабатывался по разным ценам в каждом из городов. Для каждого города будет заведен свой тарифный план.


Для решения такой задачи нужно создать «тарифную группу», в которой связать номера доступа каждого города с соответствующими тарифными планами.


1. Проверяем наличие номеров доступа в базе биллинга. Заходим в раздел “Edit Destinations”, в поле фильтра типа направления («Select type”) выбираем тип – dnis и проверяем, что все номера присутствуют.
Если номера доступа отсутствуют, их нужно добавить по правилам добавления новых направлений (указать название страны, код страны и направления и поставить тип – dnis).


2. Проверяем наличие тарифных планов для каждого из городов: Cards_Moscow (для Москвы), Cards_St_Piterburg (для Санкт-Петербурга), Cards_Izhevsk (для Ижевска).
Если некоторые тарифные планы отсутствуют, их нужно добавить, соблюдая все правила добавления новых тарифных планов.


3. В разделе “Rate groups” создаем новую «тарифную группу», в поле “New rate group” указываем имя группы и нажимаем “Add rate group”.


4. В списке “Existing rate groups” находим и выбираем новую группу.


5. Поочередно добавляем в группу тарифы для каждого из городов, предварительно выбрав для каждого из тарифов соответствующий номер доступа.



Все. Тарифная группа готова, теперь только остается назначить ее соответствующему карточному лоту, и для каждого номера доступа одни и те же карточки будут тарифицироваться в различных городах по разным тарифным планам.
