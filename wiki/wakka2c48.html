Секция привратников оператора (GK – Gatekeepers)


Для просмотра формы привратников оператора нужно выбрать пункт “gatekeepers” в меню шлюзов оператора.
Секция привратников имеет следующий вид:



Внешний вид секции привратников аналогичен виду секции шлюзов оператора.
В секции привратников присутствуют следующие кнопки управления:


КнопкаНазначение
Позволяет добавить нового привратника оператору.

Вызывает форму для редактирования параметров выбранного привратника.

Запускает процесс клонирования привратника. После нажатия на эту кнопку будет открыта форма с параметрами нового привратника, уже заполненная данными исходного привратника. После внесения требуемых изменений, форму можно сохранять. Новый привратник (измененная копия старого) будет добавлен в систему.
Следует помнить, в системе не может быть заведено двух одинаковых шлюзов.

При наведении на эту иконку курсора мыши, будет показано всплывающее окно с параметрами привратника.


При нажатии на кнопку добавления нового привратника () или редактирования имеющегося привратника () будет открыта форма с параметрами привратника, пример заполненной формы приведен ниже:



Форма состоит из следующих полей:


ПолеНазначение
Operator ipIP адрес шлюза. Поле обязательно для заполнения

IP MaskМаска входящих соединений шлюза. Параметры этого шлюза будут применимы ко всем соединениям, удовлетворяющим значениям пары address/mask. Для оригинирующих операторов есть возможность указывать подсети целиком

Пример:
Нужно задать оригинирующему оператору подсеть с адресами 192.168.0.1–255 и маской 255.255.255.0
В этом случае адресные поля должны иметь следующие значения:
Operator ip: 192.168.0.0
Ip MASK: 255.255.255.0

Gateway IDИдентификатор шлюза, имя должно быть уникальным для каждого шлюза

Proxy typeПараметр проксирования трафика (0 – проксирование только сигнального трафика, 1 – проксирование сигнального и медиа (речевого, мультимедийного) трафика

Gateway modeРежим работы шлюза. Может принимать следующие значения:
0-Ничего. Этот шлюз отключен,
1-Оригинация. С этого шлюза будет идти только оригинация трафика,
2-Терминация. С этого шлюза будет идти только терминация трафика,
3-Все. Трафик будет отправляться и приниматься с этого шлюза.
По умолчанию – 3.

Gateway typeПризнак принадлежности MVTS к зоне привратника.
1 – MVTS регистрируется на привратнике и посылает ARQ для исходящих звонков. Для входящих звонков предполагается Pregranted ARQ и привратник не информируется.
0 – MVTS запрашивает у привратника адрес назначения при помощи LRQ.
По умолчанию – 0.

IP precedenceСодержит приоритет IP-пакетов, проходящих через шлюз,
где номер – приоритет пакетов[0..7]
Необязательное поле.
Значение по умолчанию – 3 (эквивалентно операционной системе).
Примечание: Рекомендуется назначать ip_precedence выше 3 минимально возможному числу шлюзов, так как большое количество звонков с приоритетом больше 3 может вызвать нестабильность ядра операционной системы.

Proxy typeПараметр проксирования трафика.
0 – Проксирование только сигнального трафика,
1 – Проксирование сигнального и медиа (речевого, мультимедийного) трафика.
Значение по умолчанию – 0.

CapacityМаксимальное количество звонков, которое MVTS может установить с участием данного шлюза.

Priority within operatorПриоритет, согласно которому выбирается очерёдность использования ip-адресов оператора. Если у оператора несколько шлюзов, то первым будет использован тот, у которого в данном поле стоит большее число.

Dst_translateСодержат правила преобразования номера вызываемого абонента. Как правило, состоит из двух частей, разделенных символом “/”
* выражение regexp, задающее строку поиска
* строка замены
Строка поиска для удобства может быть разделена на секции символом “|”. В строке замены можно использовать подстроку, удовлетворяющую условию секции, указав номер секции после символа ”\”. Для использования в строке замены подстроки, удовлетворяющей всем секциям одновременно служит символ “&”.
Необязательное поле.

Src_translateСодержит правила преобразования номера вызывающего абонента. Как правило, состоит из двух частей, разделенных символом “/”
* выражение regexp, задающее строку поиска
* строка замены
Строка поиска для удобства может быть разделена на секции символом “|”. В строке замены можно использовать подстроку, удовлетворяющую условию секции, указав номер секции после символа ”\”. Для использования в строке замены подстроки, удовлетворяющей всем секциям одновременно служит символ “&”.
Необязательное поле.

In_dst_translateПредварительная трансляция, выполняемая при поступлении звонка от этого шлюза, перед вызовом поиска по объектам набора (dial peers).
Параметры полезны, когда какой-либо шлюз присылает номера не в формате Е164.
Правила преобразования номера идентичны правилам, приведенным в описании полей src_translate, dst_translate.

Пример 1: Терминирующий оператор просит присылать все звонки с префиксом 100#.
В этом случае, поле “Enter dst_translate” у шлюза этого терминирующего оператора должно иметь значение: [0–9]*/100#&


Пример 2: Оригинирующий оператор будет присылать вам трафик с префиксом 222#.


В этом случае, поле “Enter in_dst_translate” у шлюза этого оригинирующего оператора должно иметь значение: [0–9]*/222#&

Gatekeeper Имя привратника. Необязательное поле, однако, наличие этого поля предполагает удаление из записи о шлюзе полей port и address, во избежание нарушения логики работы программы.

Gatekeeper portПорт привратника для приема RAS трафика (по умолчанию 1719).

Gatekeeper userИмя пользователя для авторизации на привратнике оператора.

Gatekeeper passwordПароль привратника. Используется системой совместно с внутренним именем для тех же целей.

Gatekeeper securityCпоcоб авторизации на привратнике.
0 – Формирование регистрационного параметра по формуле source_alias="user|password"
1 – VocalTec
2 – CISCO
3 – CHAP
Значение по умолчанию – 0.

Gatekeeper terminalСпособ представления MVTS привратнику.
1 – Представляться привратнику как терминал,
0 – Как шлюз.
Значение по умолчанию – 0.
Необязательное поле.

Gatekeeper typeПризнак принадлежности MVTS к зоне привратника.
1 – MVTS регистрируется на привратнике и посылает ARQ для исходящих звонков. Для входящих звонков предполагается Pregranted ARQ и привратник не информируется.
0 – MVTS запрашивает у привратника адрес назначения при помощи LRQ.
Значение по умолчанию – 0.

Gatekeeper keepaliveТип сообщения, отсылаемого на привратник при перерегистрации.

Gatekeeper prefixesСписок префиксов шлюза, сообщаемый привратнику.
0 – Не предоставлять.
1 – Предоставлять список E.164 псевдонимов.
Значение по умолчанию – 0.

Change siteЕсли этот привратник нужно перенести другому оператору, нового оператора можно выбрать из данного выпадающего списка и нажать кнопку “Edit data” для переноса.

Start dateДата начала работы привратника.

Stop dateДата окончания работы привратника.

Select mera serverMVTS-сервера, на которых будет прописан этот привратник. Обязательное поле.

Additional paramsПоле для дополнительных параметров привратника. Например, для формата поддерживаемых кодеков, алертов и т.д. (Запись должна быть в формате записи MVTS-сервера).


Внизу формы расположены следующие кнопки управления:


КнопкаНазначение
Сохраняет внесенные изменения и закрывает окно редактирования.

Сбрасывает внесенные изменения и закрывает окно формы редактирования.

Удаляет редактируемый привратник оператора
Замечание: Удаление возможно только, если по данному адресу не проходил трафик в течение последних 3-х месяцев.


Действия по добавлению, редактированию и удалению привратников, аналогичны действиям со шлюзами.
