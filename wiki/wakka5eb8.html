Форма “Lots & Cards"


Основная форма для работы с карточными лотами. Здесь пользователь может создавать, редактировать и удалять карточные лоты.
Внешний вид формы «Lots&Cards» представлен на следующем рисунке:



Форма разделена на две горизонтальные части:
  • верхняя – отвечает за вывод информации по лотам;
  • нижняя – отвечает за вывод информации по карточкам в составе лота.
Выбрать необходимый лот следует из выпадающего списка «Select lot». Кнопка позволяет добавить новый лот. Параметр «Show all» позволяет просматривать закрытые лоты (по умолчанию, после того как статус лота изменяется на «закрыт», он не показывается в выпадающем списке).
После того как будет выбран лот, система загрузит информацию по всем настройкам этого лота и выведет информацию по карточкам в составе этого лота. Внешне форма «Lots&Cards» будет выглядеть следующим образом:



Описание полей секции по созданию и редактированию карточных лотов:

ПолеНазначение
Select lot 
Позволяет выбрать из ниспадающего списка для просмотра один из уже существующих лотов. Значение по умолчанию: Select lot.

Name for new lot Поле, где пользователь может вписать название лота, в процессе добавления нового лота.

DilerПозволяет закрепить за лотом дилера, который будет ответственен за реализацию карт.

DNISПозволяет выбрать из выпадающего списка номер доступа к карточной платформе.

Lot prefixПервые символы в номере создаваемых карт лота.

Pin lengthДлина пин-номера.

Pin key length Длина пин-ключа в создаваемых картах.

StateСтатус лота. Может принимать два значения: working – работает, closed – закрыт.

Charge periodПериод оплаты по карточкам. Может принимать следующие значения:
Day – позволяет установить количество дней для периода оплаты;
Month – позволяет установить количество месяцев для периода оплаты;
Year – позволяет установить количество лет для периода оплаты.
Значение по умолчанию: один год.

RateПозволяет выбрать тарифный план для использования звонков с карточек этого лота, который необходимо закрепить за данным лотом. Может принимать два значения:
Rate – позволяет закрепить за лотом выбранный тарифный план;
Rate group – позволяет закрепить за лотом выбранную тарифную группу.
Примечание: заведение тарифного плана и тарифной группы следует смотреть в разделе «Tariffs».


Surcharge shemaПозволяет закрепить за лотом схему дополнительных платежей используемого шлюза.
Примечание: схема платежей используемого шлюза настраивается в форме «Surchages».


Live period Период использования карт лота, период времени который карта будет активна.
Отсчет начинается с момента активации карты. Может принимать следующие значения:
Day – позволяет установить количество дней для периода использования;
Month – позволяет установить количество месяцев для периода использования;
Year – позволяет установить количество лет для периода использования.
Значение по умолчанию: один год.

Activation period Период активации карточек лота, период времени в течении которого можно начать звонить по карте.
Отсчет начинается с момента генерации карт, может принимать значения:
Day – позволяет установить количество дней для периода использования;
Month – позволяет установить количество месяцев для периода использования;
Year – позволяет установить количество лет для периода использования.
Примечание: значение по умолчанию активационного периода совпадает с периодом использования карт.


Start using from Позволяет установить дату начала работы лота.

Creation dateДата создания лота, значение ставится автоматически.

SubChargeСумма оплаты на балансе лота.


Описание полей секции по созданию карточек, в составе лотов:

ПолеНазначение
Cards generateСоздает карточку в составе лота. Может принимать следующие значения:
Group – создать группу карточек в составе лота;
Single – создать карточку в составе лота.

Authorization typeТип авторизации карт. Может принимать следующие значения:
PIN – авторизация по пин-коду карты;
ANI – авторизация карты по номеру звонящего абонента, с которого приходит звонок на карточную платформу,
если такого номера нет в базе системы, пойдет стандартная авторизация по PIN-у;
PIN+ANI – смешанная авторизация по пин-коду и номеру абонента, требует ввести правильный PIN с прописанного ANI.

AmountКоличество единиц на выбранной карте (его можно изменить) или количество единиц на создаваемых картах в составе лота.

ServiceТип создаваемых карт. При условии что установлен диал-ап модуль IP Studio AMBS, может принимать следующие значения:
VoIP – создать VoIP-карточку;
Dial-up (time) – создать карточку с типом Dial-up;
VoIP+Dial-up (time) – создать единую карточку и для VoIP, и для Dial-up.
Значение по умолчанию: VoIP;
WI-FI – создать карточку для беспроводного доступа в Интернет (Wi-Fi-соединение).
Примечание: Карты предназначенные для VoIP (или VoIP+Dialup) должны обязательно иметь цифровые PIN и PIN_KEY. Карты только для Dialup-а могут иметь смешанные PIN и PIN_KEY.


Cards countКоличество создаваемых карт в группе, в составе лота.
Примечание: Поле не активно при режиме «Cards generate: Single».


Pin prefixПервые цифры пинов в создаваемых картах лота.
Примечание: при режиме «Cards generate: single» поле «Pin prefix» неактивно, при создании карточки следует пользоваться полями «Manual pin», «Manual key».


Max. sessionsКоличество одновременных звонков по одному и тому же пин-коду.

Serial startНомер первой карты, с которого будут генерироваться новые карты.

CategoryТип карты. Карта бывает следующих двух типов:
Debit – Предоплаченные карты. Пользователь может звонить по таким картам до тех пор, пока баланс не дойдет до значения «Min. amount». Рассматривается только положительный баланс;
Credit – Кредитные карты. Баланс таких карт может уходить в минус.

LanguageВыбор языка в IVR.
Примечание 1: IVR (англ. Interactive Voice Response) — система предварительно записанных голосовых сообщений, выполняющая функцию маршрутизации звонков внутри call-центра, пользуясь информацией, вводимой клиентом с помощью тонального набора.

Примечание 2: Редактирование созданной карточки доступно в форме «Card statistic», пользователь может перейти в эту форму путем клика на ссылку в поле «Card number».




Описание полей секции по выводу информации по карточкам в составе лота:

ПолеНазначение
FilterОсуществляет поиск карточки в составе лота по различным параметрам выборки:
PIN – осуществляется поиск по пин-коду карты. Доступно два режима:
  • Number – осуществляется поиск по отдельно взятому пин-коду карты;
  • Range – осуществляется поиск по указанному диапазону пин-кодов карт.
Card number range – осуществляется поиск по названию карточки. Доступны два режима:
  • Number – осуществляется поиск по отдельно взятому названию карты;
  • Range – осуществляется поиск по указанному диапазону названия карт.

ANI – поиск карты по телефонному номеру, который используется во время авторизации карты.

First name – поиск карты по имени пользователя, в случае если карточка является персональной.
Second name – поиск карты по фамилии пользователя, в случае если карточка является персональной.
Organization name – поиск карты по названию организации, в случае если карточка является персональной.

Amount – поиск по количеству единиц на балансе карточек. Доступны режимы:
  • More – будут выводиться карточки, у которых количество единиц на балансе больше указанного значения;
  • Less – будут выводиться карточки, у которых количество единиц на балансе меньше указанного значения;
  • Equals – будут выводиться карточки, у которых количество единиц на балансе равно указанному значению.

Status – поиск осуществляется по статусу карточек. Доступны режимы:
  • New – будут выводиться не активированные карточки;
  • Active – будут выводиться активированные карточки;
  • Blocked – будут выводиться заблокированные карточки;
  • Closed – будут выводиться закрытые карточки.

Change status – изменить статус карты (на Active или Blocked), доступны режимы:
  • Selected cards – выбранные карточки;
  • All cards – все имеющиеся карточки в составе лота;
  • Filtered cards – все карточки, полученные в результате выборки по параметрам поиска.

ExportПозволяет сохранить информацию по карточкам в файл формата .csv.

Transfer cardsПозволяет перенести карточку или группу карточек в состав другого лота. Содержит два поля:
Select lot  – позволяет выбрать необходимый лот, в который требуется перенести карточки.
Selected cards – позволяет перенести выбранные карточки в состав другого лота. Доступно два режима:
  • Selected cards – переносит карточки в состав другого лота, выделенные вручную путем проставления галочек;
  • Filtered cards – осуществляет перенос карточек, полученные по критериям поиска в поле «Filter».


Views rowsКоличество выводимых строк, которые будут показаны на дисплее ПК за один раз. Может принимать следующие значения: 10, 20, 30, 50, 100 или All (все имеющиеся).
Значение по умолчанию – 20.

Root certПоле, позволяющее установить корневой сертификат для wi-fi-карточек – get root certificate, путем перехода по ссылке «get root certificate».
Примечание: Сертификат – цифровой документ, подтверждающий соответствие между «открытым ключом (root certificate)» и информацией, идентифицирующей владельца ключа. Содержит информацию о владельце ключа, сведения об открытом ключе, его назначении и области применения, название центра сертификации и т. д.
Открытый ключ используется для организации защищенного канала связи с владельцем двумя способами:
• для проверки подписи владельца (аутентификация);
• для шифрования посылаемых ему данных (конфиденциальность).



В форме присутствуют следующие кнопки управления:

КнопкаНазначение
Add lot. Вызывает форму по созданию нового лота.

Edit lot. Вызывает форму для редактирования параметров лота.

Card generation. Вызывает форму по созданию карточек в составе лота.

Import cards. Позволяет загрузить файл с данными карточек в состав лота.

Find. Осуществляет поиск карточек по установленным параметрам выборки.

Change status. Изменяет статус карты или группы карточек.

Export to excel. Сохраняет информацию по карточкам в файл формата .csv.

Transfer cards to a selected lot. Осуществляет перенос карточек в выбранный лот.
