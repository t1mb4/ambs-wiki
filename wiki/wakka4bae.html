Секция параметров оператора



В этой секции показаны все параметры работы с оператором и его контактная информация. Каждый оператор в системе AMBS имеет следующие параметры:


ПолеНазначение
Operator descriptionОписание оператора. Сокращение или аббревиатура названия компании оператора

Organization nameПолное название компании

AddressАдрес компании оператора

PhonesКонтактные телефоны компании

FaxФакс компании оператора

Agent orig/term (%)Процент с оригинации/терминации, который будет получать агент, работающий с этим оператором

ManagerНазначенный данному оператору менеджер, который ведет этого оператора в процессе работы

Fin. contact personКонтакт для финансовых вопросов

Tech. contact personКонтакт для технических вопросов

CompanyКомпания, от лица которой работаете с этим оператором

Invoice parametersПараметры напоминания о выставлении счета, бывают нескольких типов:
Auto invoice – Автоматическое напоминание. Отправляет напоминание на электронный адрес менеджера, курирующего этого оператора, о сроке выставления счета оператору
Mail invoice – Создается и отсылается готовый инвойс оператора на электронный адрес менеджера
Mail detalization – Создается и отсылается CDR-файл с детализацией звонков за прошедший период на электронный адрес менеджера

Invoice periodПериод, как часто система будут отправлять менеджеру напоминания, счета или CDR-ы данного оператора:
Every day – Каждый день
Every week – Каждую неделю
Every month – Каждый месяц
2 times per month – Два раза в месяц

Max creditДоступный оператору кредит.
Если данный параметр имеет нулевое значение, оператор имеет неограниченный кредит.
Если задана сумма больше нуля, система будет принимать звонки оператора до тех пор, пока баланс оператора не уйдет в минус до установленной суммы.
Пример: Установлен баланс 100$, это значит, что система будет принимать звонки оператора до тех пор, пока баланс оператора не станет -100$.

Time differenceРазница между часовыми поясами операторов. По умолчанию имеет значение – 0. Если выставлена любая другая величина, то эта разница будет автоматически использоваться при создании инвойсов

Sort routes by…Определяет, по какому фактору выбирать список доступных направлений для этого оригинирующего оператора: по минимальной цене, по ASR, по ACD или по индексу качества Qi

Minimal percentПоле, отвечающее за прибыль от этого оператора. Может иметь отрицательное значение

Operator statusТекущий статус оператора:
Working – Оператор работает
Closed – Оператор закрыт

Q931 codeТип кода, который будет отсылаться этому оператору, если его звонки не будет отбивать сама система. Например, если оператор работает по кредитной системе и у него закончился кредит

Operator typeТип оператора, определяет его статус в системе и уровень его доступа.
Существует несколько типов операторов:
Partner – Оператор, с которым работают непосредственно
Retail – Клиент оператора типа “Partners” имеющий свои тарифные планы в системе
Virtual – Виртуальный клиент для оператора типа “Retail”. На данный момент не используется в системе

Default currencyВалюта в которой работают тарифные планы оператора. Должна совпадать с валютой выбранной во всех тарифных планах оператора.
По правилам системы один оператор может работать только в одной валюте.

Use in routingПредназначена для включения тарифов оператора в работу по тайм зоне оператора.
Например, удобно при работе между российским и американским операторами, поскольку разница во временных зонах между Россией и Америкой 8–10 часов.
При сверке один и тот же звонок может у одной стороны тарифицироваться по новой цене у другой стороны еще по старой, данная функция позволяет учитывать данный момент.


Для управления формой параметров оператора внизу формы расположены следующие кнопки управления:


КнопкаНазначение
Вызывает форму для редактирования параметров операторов

Позволяет добавить комментарии

Показывает все комментарии

Добавляет клиента в систему для выбранного оператора

Секция АОН-групп. За оператором можно закрепить несколько групп (Group) с наборами(Set) АОН-номеров.
В форме “ANI Reports” можно получить статистику по группе или набору номеров закрепленным за оператором.

Удаляет выбранного оператора. Чтобы удалить оператора, за ним не должно быть закреплено сайтов и шлюзов.
Если они присутствуют, сначала нужно удалить их.
