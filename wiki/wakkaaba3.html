Форма “MVTS Servers”


Форма для подключения к АМБС новых MVTS, SIP и других серверов, которые будут работать под управлением IP Studio AMBS.
Внешний вид формы показан на следующем рисунке:



Поля формы:

ПолеНазначение
Show disabled serversВыводи на экран сервера со статусом “Closed”.
По умолчанию такие сервера не показываются.

Server nameИмя сервера, под которым будет заведен в системе.

Server IPIP подключаемого сервера

Radius IPIP сервера с радиусом, куда будет обращаться подключаемый сервер

Radius auth portПорт авторизации на радиусе
Значение по умолчанию – 1812

Radius acct portПорт аккаунтинга на радиусе
Значение по умолчанию – 1813

Route port:Порт роутинга радиуса
Значение по умолчанию – 1814

Local auth portЛокальный порт авторизации радиуса
Значение по умолчанию – 1645

Local acct portЛокальный порт аккаунтинга радиуса
Значение по умолчанию – 1646

Local route port:Локальный порт роутинга радиуса
Значение по умолчанию – 1647

Radius keyКлюч радиуса (secret key). Используется для обмена информацией с сервером.
Значение должно быть одинаковым: в форме серверов, в файле radius.cfg
и в конфигурационном файле сервера, для MVTS-серверов это meraproxy.cfg .
Значение по умолчанию – testing123

FTP loginИмя пользователя(ftp/sftp) используемое системой для подключения к удаленному серверу.
Поле обязательное к заполнению.
Значение по умолчанию – ipstudio

FTP passwordПароль(ftp/sftp) используемый системой для тех же целей.

Server stateСтатус сервера (работает или отключен)
Отключенные сервера не будут показываться в списке заведенных серверов по умолчанию.
Значение по умолчанию – Enabled(Включен)

File transfer protocolТип протокола передачи данных (ftp/sftp) при обращении к серверу (используется, если заводится удаленный сервер)
Рекомендуется использовать – sftp

SNMP versionИспользуемая версия SNMP для работы с Infinet-ом.
Значение по умолчанию – SNMP v1

SNMP groupSNMP-Группа для работы с Infinet-ом
Значение по умолчанию – public

SNMP Authorization PasswordПароль для подключения к Infinet-у, для чтения его конфигурации.

SNMP Authorization protocolПротокол авторизации для считывания конфигурации с Infinet-а.
Значение по умолчанию – md5

SNMP Private Password Пароль для подключения к Infinet-у, для записи в него конфигурации.

SNMP Private protocolПротокол авторизации для записи конфигурации на Infinet.
Значение по умолчанию – des

Server typeТип добавляемого сервера.
Значение по умолчанию – MVTS

Server protocolПротокол работы сервера
Значение по умолчанию – Н323
Примечание: Для MVTS-серверов это поле не используется. Рекомендуем оставлять значение по умолчанию.

Monitor portПорт монитора Infinet-а.

Monitor passПароль от монитора Infinet-а.


Кнопки управления формы:


КнопкаОписание
Add serverДобавляет новый сервер в базу системы, после заполнения всех полей параметров этого сервера

Edit propertiesОбновляет и сохраняет информацию о выбранном сервере, после ее исправления

Delete serverУдаляет выбранный из списка сервер


После того, как все поля нового сервера будут заполнены, необходимо нажать кнопку “Add server” внизу формы для окончания процесса создания.
Если все поля были заполнены верно, новый сервер будет добавлен в лист подключенных серверов, работающих под управлением системы IP Studio AMBS:



Для того чтобы произвести какие-либо действия над серверами в этом списке, предварительно требуемый сервер нужно выбрать, поставив точку в первом поле напротив его имени.
Ниже списка доступных серверов расположены следующие кнопки управления:


КнопкаОписание
Copy gatesКопирует все заведенные в АМБС шлюзы с выбранной из списка серверов меры на указанный в выпадающем списке (напротив кнопки).

Update gateway.cfgОбновляет конфигурационные файлы на выбранном в списке сервере

Enable/Disable RadiusВключает/выключает режим «Авторизации и Маршрутизации» для выбранного сервера.
При этом в файле “meraproxy.cfg” этого сервера система добавит/изменит секцию “Radius”
Секция радиуса должна идти последней в конфигурационном файле MVTS.

Enable/Disable AccountingВключает/выключает режим «Аккаунтинга» для выбранного сервера. При этом в файле “meraproxy.cfg” этого сервера система изменит значение параметра “Enable Accounting” в секции “Radius”
Секция радиуса должна идти последней в конфигурационном файле MVTS.

Load gateways list from fileЗагружает все ip-адреса для выбранного сервера из указанного файла “gateway.cfg”.
Внимание, если какие то шлюзы уже присутствуют в системе для этогосервера(были заведены в ручную) они будут перезаписаны новыми загружаемыми данными.
Примечание: Будут загружены только ip шлюзов, без других дополнительных параметров.

Show ip without operatorsПоказывает адреса шлюзов присутствующие в системе, но еще не назначенные операторам.

Show mera reload statusПоказывает лог последнего обновления серверов.


При добавлении нового сервера нужно обязательно учитывать, какой тип протокола передачи данных будет использоваться для обмена данными с этим сервером.
– для использования ftp-протокола, ваши сервера должны поддерживать подключение по ftp-протоколу;
– протокол “sftp” более безопасный, но для его использования ваши сервера должны поддерживать ssh-подключение;
– если MVTS расположен на том же самом сервере где и AMBS, тип протокола не имеет значения, все происходит локально.


Примечание: После заведения нового mvts-сервера нужно в форме Операторов добавить оператору “Unknown IP” шлюз “0.0.0.0” на этой мере.
Это делается для того, чтобы все звонки которые система не смогла отправить получателю складывались одному общему оператору.
Упростит поиск причин неисправностей в настройках в дальнейшем.
