Примеры работ с лотами


1. Пример заведения лота.
Например, создадим лот с учетом следующих условий:

  • Название лота – Test_lot;
  • Название карт должно начинаться с символов – vt;
  • Длина пина – 6 символов;
  • Длина пин-ключа – 6 символов;
  • Лот должен быть рабочим в течение одного года.

1. Следует открыть форму по созданию нового лота, путем нажатия кнопки «Add lot»;
2. Вписать название лота «Test_lot» в поле «Name for new Lot»;
3. Вписать префикс лота «vt» в поле «Lot prefix»;
4. Указать длину пина и пин-ключа (6 символов) карт лота в соответствующих полях «Pin length» и «Pin key length»;
5. Даты сроков активации карт лота и периода работы карт лота – поставить 1 год;
6. Нажать кнопку «Add Lot».



Примечание: Для расчета баланса карточек в составе лота необходимо выбрать соответствующий тарифный план (тарифную группу) в поле «Rate/Rate group».

2. Пример блокирования лота.
Например, необходимо заблокировать выше созданный лот «Test_lot», следует выполнить следующие действия:

1. В поле «Select lot» выбрать название лота «Test_lot»;
2. Нажать кнопку «Edit lot» для открытия формы по редактированию созданного лота;
3. В поле «State» выставить параметр «Closed»;
4. Нажать кнопку «Apply change» для подтверждения внесенных параметров.



Примечание: Что бы найти заблокированный лот, пользователю необходимо воспользоваться параметром «Show all» в поле «Select lot».

3. Пример по смене закрепленного тарифного плана на группу тарифных планов.
Например, один карточный лот используется в нескольких городах, регионах. Звонки приходят на карточную платформу по разным номерам доступа и звонки из каждого города необходимо тарифицировать по тарифным планам этих городов. Для того чтобы такая схема работала, следует назначить карточному лоту группу тарифных планов, распределенных по номерам доступа(DNIS).
Для того, что бы закрепить за определенным лотом тарифную группу вместо тарифного плана, необходимо выполнить следующие действия:

1. Создать группу тарифных планов (название группы – Test), описание и заведение группы тарифных планов см. в форме «Rate Groups»;
2. В поле «Select lot» выбрать название лота «Test_lot»;
3. Нажать кнопку «Edit lot» для открытия формы по редактированию лота;
4. В поле «Rate/Rate group» поставить параметр «Rate group»;
5. В выпадающем списке выбрать тарифную группу, которую необходимо закрепить за лотом;
6. Нажать кнопку «Apply change» для подтверждения внесенных параметров.


4. Пример добавления номера доступа (DNIS) в лот.

1. Проверить наличие номеров доступа в базе биллинга. Для этого следует зайти в форму «Edit Destinations», в поле фильтра типа направления («Select type») выбрать тип «dnis» и проверить, что все номера присутствуют. Если номера доступа отсутствуют, их следует добавить по правилам добавления новых направлений (указать название страны, код страны и направления и поставить тип «dnis»). Для данного примера DNIS – Moscow;

Примечание: Добавление и удаление новых направлений на примере можно посмотреть в форме «Edit destinations».

2. В поле «Select lot» выбрать название лота «Test_lot»;
3. Нажать кнопку «Edit lot» для открытия формы по редактированию лота;
4. В поле «DNIS» из выпадающего списка выбрать номер доступа – Moscow;
5. Нажать кнопку «Apply change» для подтверждения внесенных параметров.


5. Пример блокирования группы карточек в составе лота.
Если необходимо заблокировать группу карточек в составе лота – это можно осуществить следующим образом:

1. В поле «Filter» выбрать значение «Card number range»;
2. По выбранному значению «Card number range» поставить параметр «Range»;
3. Ввести названия карт (начальное и конечное) для диапазона карт, которые необходимо заблокировать;
4. Нажать кнопку «Find» для осуществления поиска по указанному диапазону;
5. В поле «Change status» выбрать значение «Filtered cards» и поставить параметр «Blocked»;
6. Нажать кнопку «Change status» для подтверждения блокировки карт.
