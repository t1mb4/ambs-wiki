Форма “Configuration”


Форма позволяет производить настройки файла конфигурации “billing/etc/bill_util.cfg” из интерфейса системы. Состоит из следующих полей:

ПолеНазначение
Database parameters
DATABASEНазвание базы данных системы. Не рекомендуется менять значение по умолчанию.

HOSTАдрес хоста, на котором расположена система IP Studio AMBS.

PORTПорт, по которому система будет подключаться к базе данных.

USERПользователь MySql. Используется системой для работы с базой.

PASSПароль для пользователя. Используется системой совместно с внутренним именем для тех же целей.

System path parameters
LOGПуть к лог-файлу карточного модуля.

PARAM_FOR_TARIFФайл для сохранения параметров операции “Recharge”.

TARIF_SCRIPTПуть к скрипту запуска операции “Recharge”.

TARIF_OUTPUTФайл для сохранения итоговых результатов операции “Recharge”.

LOCKПуть к файлу системных параметров радиуса системы.

PIDFILEID-файл радиус-процесса.

LOG_FILEПуть к файлу логов системы IP Studio AMBS

PREFIXПуть к папке расположения системы на сервере.

MVTSПуть к директории, где расположен MVTS-сервер.

CDRDIRПуть к директории, где располагаются CDR-файлы MVTS-сервера.

BINDIRПуть к директории расположения исполняемых скриптов MVTS-сервера.

RECALC_SCRIPTПуть к скрипту запуска операции “Recalculation”.

BACKUP_TMP_FILEПуть и имя временного файла для резервного копирования логов прошедших парсинг.

NOBACK_CDRНе делать резервную копию CDR-ов после их парсинга.
0 – делать копию.
1 – не делать копию.
Значение по умолчанию – 0.

MYSQLПуть к консоли MySql.

CDR_PREFПрефикс CDR-файлов Меры.
Значение по умолчанию – bill.

DEBUGВключает режим вывода на экран технической информации.

Server parameters
MY_IPАдрес сервера (на котором редактируется конфигурационный файл).

ServPortПорт демона системы.

LASTFILEФайл с датой и временем последней сессии парсинга логов MVTS-сервера.

BILLINGРежим работы системы:
Yes – Режим биллинга, полнофункциональный
No – Режим демона, работает только демон
Значение по умолчанию: Yes

OWNERСистемный владелец системы IP Studio AMBS
Значение по умолчанию: ipstudio
Не рекомендуется изменять значение по умолчанию, поскольку под этим же пользователем работают apache и mysql, возможен сбой в работе в системе.

CONN_STRСтрока подключения для обновления системы.

BUFFERПараметр буферизации:
1 – с буферизацией, для очень больших объемов трафика, информация будет поступать в систему не сразу, а с некоторой задержкой
0 – без буферизации, вся новая информация будет поступать в систему сразу
Значение по умолчанию: 0

MAX_ROUTESЛимит роутов выдаваемых Мере радиусом.
Не рекомендуем ставить больше 20.
Значение по умолчанию – 9.

MAX_GWSМаксимальное количество роутов одного оператора выдаваемое Мере.

Mail parameters
MAIL_SERVERАдрес почтового сервера для отправки писем из системы.
По умолчанию, система использует установленный на сервере “sendmail”.
Замечания:
1. Работа почты в системе полностью зависит от работоспособности sendmail-а.
2. Почтовый модуль системы не поддерживает авторизацию.

ALERT_MAILОтправитель, для писем системы.
Значение по умолчанию: alert@ipstudio.net

INV_SEND_FMTФормат инвойсов отправляемых из раздела “Mail invoices”
Может принимать следующие значения:
tar – инвойсы будут приходить в виде HTML-страницы в архиве
xls – инвойсы будут приходить в виде Excel-страницы

SEND_CONFIRM_REGОтправлять подтверждение после заведения нового пользователя.
Routing parameters
Qi = (QI_ASR*(Compl_calls,%) + QI_ACD*(ACD*10) + QI_SH*(100 – Sh_calls,%)) / 3
BIG_CODESПараметр использование более длинных кодов в первую очередь при построении списка направлений для маршрутизации звонка.
1 – Более длинные коды будут использоваться в первую очередь
0 – Длина кодов учитываться не будет
Значение по умолчанию: 1

QI_ASRКоэффициент ASR. Используется в формуле расчета Qi.

QI_ACDКоэффициент ACD. Используется в формуле расчета Qi.

QI_SHКоэффициент Short Calls. Используется в формуле расчета Qi.

STAT_HOURSПериод времени в часах через который будут обновляться показатели качества (для их применения в расчете Qi).

MIN_CALLS_CNTМинимальное количество звонков прошедших на оператора для расчета Qi этого оператора.
Если звонков было меньше, оператору будет назначен Qi по умолчанию.

DEF_ASRЗначение ASR по умолчанию. Используется для расчета Qi по умолчанию.

DEF_ACDЗначение ACD по умолчанию. Используется для расчета Qi по умолчанию.

DEF_SHЗначение Short Calls по умолчанию. Используется для расчета Qi по умолчанию.

DELIMРазделитель для прямых звонков на выбранного оператора по ID его сайта.

Money parameters
RETAIL_DEF_AMOUNTИсходное значение баланса для розничных клиентов Астериска.

REMAIN_PROCМинимальный процент кредита для прохождения звонков через Астериск.

ASTSIP_DEF_CONTEXTИсходный контекст для Астериска.

DEF_VALUTEБазовая валюта системы.

KURS_SCHEMEСхема обновления кусов валют в системе (локально в ручную или с сайтов банков).

Netflow parameters
NFDIRПолный путь до директории где установлен Netflow

NF_PATTERNПрефикс файлов Netflow
Значение по умолчанию – ft*

NF_RMУдалять файлы Netflow после их обработки или нет.

NF_DEL_FILERECОтчищат журнал ззагруженых файлов Netflow.

GRP_NF_PORTSГруппы портов Netflow

Memcache parameters
memcache_useСтатус memcache-а.
1 – включен
0 – выключен
Значение по умолчанию – 1

memcacheПолный путь до memcache-а.

memcache_sizeОбъем memcache-а.

memcache_ipАдрес сервера на котором запущен memcache.

memcache_portСерверный порт используемый memcache-ем.


Внизу формы расположены кнопки управления. Кнопка “Set modify” позволяет сохранить внесенные изменения. Кнопка “Set default” – устанавливает исходные значения системы.
