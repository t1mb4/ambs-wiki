Секция направлений тарифного плана


После заведения нового тарифного плана или при открытии уже существующего плана, на экране будет показан лист с направлениями выбранного плана.



Первая строка: “This rate plan is assigned to ….” показывает, к сайтам каких операторов прикреплен выбранный тарифный план. Например, на рисунке тестовый тарифный план закреплен за оператором “Test123” как оригинационный тариф.

Примечание: Один тарифный план можно закрепить за несколькими операторами и при этом у некоторых из них он может быть закреплен как терминационный тарифный план, у других как оригинационный план. Нужно быть внимательным. При изменении цены на любое из направлений в таком тарифном плане изменения скажутся на всех операторах, за которыми закреплен этот план.

Следующая строка показывает количество страниц с направлениями, в тарифном плане и какая из них на данный момент показана на экране. Количество выводимых за один раз на экран строк регулируется параметром “Rates list rows count”. Если количество строк в плане больше, чем показано за раз на экране, то тарифный план можно пролистать с помощью стрелок, расположенных в данной строке.

Далее идет лист с направлениями в виде списка с кодами направлений, их названиями и основной информацией о каждом из направлений.

Внизу списка расположены контрольные кнопки для работы с направлениями в тарифном плане.

Для каждого направления в листе показаны следующие поля:

ПолеНазначение
Enable(-)Поле активации выбранной строки направления. После активации параметры “Price”, "Start/Stop dates" и “Close” строки направления будут доступны для редактирования.

DestinationНазвание направления.

PriceЦена направления.

Start dateДата начала работы направления. Тарифное направление считается рабочим с первой секунды этой даты.

Stop dateДата окончания работы тарифного направления. Направление становится нерабочим.
Тарифное направление считается нерабочим с первой секунды этой даты. После этой даты звонки на это направление по этой цене проходить не будут.
Если стоп дата совпадает со старт датой, это направление считает – нулевым (начало работы в 00:00 и завершение в этот же самый момент 00:00).

Code stringСписок кодов, входящих в данное направление. Поле имеет тип – Text, т.е. количество кодов в строке не ограничено. Объем кодов будет влиять на скорость открытия страницы с тарифным планом при просмотре.

CloseЗакрыть направление. Позволяет временно закрыть данное направление. Если в этом поле стоит галочка, то звонки на это направление проходить не будут.

RoundПараметры округления для направления.

Time zoneПериод времени, по которому работает направление.

Примечание: Если нужно, чтобы одно направление работало в нескольких периодах времени, эти записи нужно загружать отдельными файлами. Один код может встречаться в файле загрузки только один раз.


Min/Unpaid sec.Дискретность и неоплачиваемые секунды направления.

Всплывающее информационное окно. Содержит детальное описание всех параметров направления.


Также для каждого в направления в форме присутствует кнопка редактирования направления при ее нажатии будет открыто окно с параметрами направления для их редактирования.



Каждое направление в тарифном плане имеет следующие параметры:

ПолеНазначение
DestinationНазвание направления.

PriceЦена направления.

Start dateДата начала работы направления. Тарифное направление считается рабочим с первой секунды этой даты.

Stop dateДата окончания работы направления. Тарифное направление считается нерабочим с первой секунды этой даты. После этой даты звонки на это направление по этой цене проходить не будут.

CloseЗакрыть направление. Позволяет временно закрыть данное направление. Если в этом поле стоит галочка, то звонки на это направление проходить не будут.

GroupГруппа направлений, к которой будет принадлежать данное направление. Используется для настройки маршрутизации на основе групп направлений операторов.

Deny groupЗапрещенная группа направлений, куда (откуда) звонки по этому направлению от (на) данного оператора проходить не будут. Используется для настройки маршрутизации на основе групп направлений операторов.

Close hourПериод закрытия в часах, на сколько будет закрыто данное направление. После истечения этого срока направление автоматически будет открыто и галочка в поле “Close” снята.

MinsecМинимальная продолжительность звонка. При длительности звонка меньше указанного в этом поле периода секунд, звонок будет считаться бесплатным.

Unpaid secНе оплачиваемые секунды. Количество бесплатных секунд с начала разговора.

Discr.Дискретность. Количество секунд составляющее единицу цены. По умолчанию установлено значение – 60 секунд, т.е. единица цены (указанная в поле “Price”) будет сниматься за каждые 60 секунд разговора.

First roundМинимальная длительность звонка. Если длина звонка меньше периода секунд в этом поле, длительность звонка будет приравнена к указанному здесь периоду.

RoundОкругление длительности разговора. По умолчанию устанавливается 1 секунда, т.е. посекундная тарификация.

Num countФильтр длинны номера на данное направление. В поле указывается точная допустимая длина номера на это направление. Например, если все номера должны состоять из 11 цифр, то можно выставить в этом поле значение “11” чтобы все неправильные номера отбивались.

CapacityДопустимое количество одновременных звонков по направлению. Значение по умолчанию – 255. При необходимости можно ставить большее или меньшее значение.

Time zoneПериод времени, по которому работает направление.


Внизу формы параметров расположены кнопки управления:

КнопкаНазначение
Сохраняет внесенные изменения и закрывает окно параметров

Сбрасывает внесенные изменения и закрывает окно формы


Теперь рассмотрим кнопки управления направлениями в тарифном листе, расположенные внизу листа.



Кнопки позволяют производить операции как с одним направлением в листе, так и с группой.

Кнопка/ПолеНазначение
Change pricesДобавляет в тарифный план направление(я) на основе имеющихся, но с изменениями цены и периода работы.
Одновременно можно менять одно или несколько направлений.
Чтобы добавить в тарифный план новое направление (копию уже имеющегося, но с новыми параметрами) нужно у старого направления изменить цену и дату начала работы, чтобы они учитывались в новом направлении.
Рекомендуем ознакомиться с работой данной функциональности в разделе: Примеры работы с направлениями в тарифном плане.

Change prices without historyСохраняет изменения, внесенные в параметры направления. Не добавляет и не изменяет другие направления. Используется для исправления ошибок в направлениях.
Важно: При изменении цены на направление, нельзя просто изменять его цену и старт дату с помощью этой кнопки. В этом случае все звонки, уже прошедшие на данное направление по старой цене в течение старого периода, станут не калькулированными, поскольку для них будет отсутствовать направление и цена в тарифном плане. Нужно добавить новое направление с новой ценой и исправить периоды работы направления по разным ценам или воспользоваться функцией – Change price.

Delete pricesУдаляет активированные направления (направления с галочкой в первом поле Enable(-)).

Check AllАктивирует все направления, показанные на экране.

Uncheck AllОперация обратная предыдущей операции. Делает все направления неактивными для редактирования.

Close rate time (hours)Количество часов, на которое будут закрыты направления.
Можно установить для:
Close All – Закрывает на установленное время все направления в тарифном плане.
Close selected – Закрывает все выбранные(активированные) направления на установленный период времени.
Close filtered – Закрывает все направления показанные с помощью системы фильтров.
Open all – Открывает все выбранные(активированные) направления.

Set Start/Close dateУстанавливает дату открытия или закрытия для направлений.
Дата должна иметь формат системы: ГГГГ-ММ-ДД
Можно установить дату для:
Set for all active – Устанавливает дату для всех активных направлений в тарифном плане (все направления показываемые при фильтре – current).
Set for selected – Устанавливает дату для всех выбранных направлений (направления с галочкой в первом поле – Enable).
Set for filtered – Устанавливает дату для всех направлений, выбранных с помощью системы фильтров.

Set Group/Deny groupПоля для установки группы и запрещенной группы, которые будут назначены направлениям.
Используется для настройки маршрутизации на основе групп направлений операторов (Route groups).
Можно установить для:
Set for all active – Назначает выбранные группы всем активным направлениям в тарифном плане (все направления, показываемые при фильтре – current).
Set for selected – Назначает выбранные группы всем выбранным направлениям (направления с галочкой в первом поле Enable(-)).

Set Time zoneУстанавливает выбранный период времени для направлений.
Можно установить для:
Set for all active – Назначает выбранный период всем активным направлениям в тарифном плане (все направления, показываемые при фильтре – current).
Set for selected – Назначает выбранный период всем выбранным направлениям (направления с галочкой в первом поле Enable(-)).

First round/Round/Discr.Устанавливает параметры округления и дискретности для направлений.
Можно установить для:
Set for all active – Назначает параметры всем активным направлениям в тарифном плане (все направления, показываемые при фильтре – current).
Set for selected – Назначает параметры всем выбранным направлениям (направления с галочкой в первом поле Enable(-)).

Excel versionПозволяет экспортировать выбранные направления из тарифного плана в CSV-файл.


Важно: При работе с CSV-файлами в Excel следует помнить. При открытии CSV-файла через функцию «Открыть файл» в Excel, последний будет автоматически подставлять «Обычный» формат ячеек для всех загружаемых данных. При использовании такого типа ячеек, автоматически будут убираться все нули стоящие в начале данных.

Например, код Москвы – 095, будет показан как – 95, получаем код – 795, а это уже совершенно другое направление. Также возможны ситуации с авто подменой цифровых данных на даты.

Чтобы избежать подобных ошибок при открытии CSV-файлов в Excel, рекомендуем использовать функцию «Импорт внешних данных» (Данные -> Импорт внешних данных -> CSV-файл). При загрузке данных через эту функцию нужно будет ответить на серию стандартных вопросов о типе загружаемых данных («с разделителями» – разделитель «точка с запятой») и на последнем шаге указать, что все импортируемые данные должны быть загружены в текстовом формате.
