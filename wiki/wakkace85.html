Форма “Route Groups”


Здесь можно организовать группировку направлений оригинаторов и терминаторов. Другими словами, создать группу с набором направлений оригинаторов и терминаторов (или их сайтов целиком) звонки через которые будут иметь приоритет внутри группы, т.е. звонки оригинаторов, по направлениям входящим в группу, будут первым выбором идти на доступные направления терминаторов входящих в эту же группу, и уже потом на все остальные доступные направления по стандартным правилам маршрутизации.
Форма имеет следующий вид:



Поля формы:


ПолеНазначение
Select groupСписок всех заведенных в системе групп.

Group nameПоле для имени группы, для изменения имени текущей(выбранной) или создания новой группы.

Transit flagВключает функцию транзита трафика между операторами внутри группы.
По отдельным транзитным ценам "Carrier/Client prices" заданным здесь же в форме.
Обычно используется для группы из одного направления для транзита трафика между двумя операторами.

Client priceЦена оригинатора для транзита.

Carrier priceЦена терминатора для транзита.


Кнопки формы:


КнопкаНазначение
DeleteУдаляет выбранную группу.

EditИзменяет имя выбранной группы на новое, указанное в поле “Group name”.

AddСоздает новую группу с именем, указанным в поле “Group name”.


Всего существует два типа групп:


ПолеНазначение
GroupГруппа принадлежности. Направления и сайты внутри одной группы будут иметь первоочередной приоритет при маршрутизации.

Deny groupЗапрещенная группа. Трафик группы никогда не пойдет на направления (сайты) заданной запрещенной группы.


Рабочие поля группы имеют следующий вид:



Визуально, форму группы можно разделить на несколько секций по вертикали:

  • Секция выбора групп
  • Секция вложенных групп
  • Секция работы с сайтами операторов
  • Секция информации по направлениям операторов.

Рассмотрим каждую из перечисленных секций:


1. Секция выбора групп.


Здесь можно выбрать группу направлений для работы с ее деталями, создать новую группу или изменить имя существующей.


2. Секция вложенных групп.


В любую группу можно добавить «вложить» другие группы направлений. В этом случае, правила, заданные в текущей группе будут распространяться навложенные группы.
Правила заданные во вложенных группах по прежнему будут работать внутри этих групп.


3. Секция работы с сайтами операторов.


Секция позволяет добавлять сайты в группу целиком. Из добавленных сайтов будут браться те тарифный план, который подходят для работы секции куда добавили сайт.
Если сайт добавлен в секцию оригинирующих сайтов, будет использоваться оригинирующий тариф закрепленный на этом сайте, если в секцию терминирующих сайтов, будет использоваться – терминирующий тариф.
Всего есть 4 правила разделенные на 2 части:

  • Included sites – Разрешенные сайты, входящие в состав группы;
    • Origination side – Оригинационные сайты, оригинирующий трафик с этих сайтов будет являться частью группы;
    • Termination side – Cайты терминации, будут идти первым приоритетом для трафика этой группы;

  • Deny sites – Запрещенные сайты, исключенные из маршрутизации данной группы;
    • Selected originators sites cannot send traffic to this group – Запрещенные оригинационные сайты, трафик с указанных сайтов никогда не пройдет на терминирующие сайты этой группы;
    • Selected terminators sites would not accept traffic from this group – Заблокированные сайты терминации, трафик группы никогда не будет попадать на указанные здесь сайты;

Примечание: Параметр “this group only” разрешает отправку/прием трафика только внутри данной группы.

4. Секция информации по направлениям операторов.
Показывает сколько направлений в каком тарифном плане оператора назначенно в текущую группу.
Секция также поделена на 4 правила (2 части: разрешенные направления и запрещенные направления):

  • Included routes – Разрешенные направления.
    Origination routes – Направления оригинации входящие в данную группу.
    Termination routes – Направления терминации входящие в данную группу.
  • Deny routes – Запрещенные направления.
    These originators routes cannot send traffic to this group – Заблокированные направления оригинации, не могут слать трафик на терминирующие направления этой группы.
    These termination routes cannot receive traffic from this group – Заблокированные направления терминации, трафик группы не будет проходить на эти направления.

Назначение направлений в группу производится в форме “Edit Tariffs”. Для этого нужно выбрать тарифный план, выбрать направления и назначить их в нужную группу как принадлежащие группе(“Group”) или как запрещенные для группы (“Deny group”).


Важно: Особо следует отметить приоритетность выбора направлений терминации для направления, включенного в группу:

1. Направления, заданные вручную в разделе “Operators route parameters” для данного направления оригинации,
2. Терминационные направления из этой же или вложенных групп,
3. Остальные операторы, предлагающие данное направление, отсортированные с учетом параметра “Sort routes by” оригинатора.
4. Из списка будут исключены операторы, направления и сайты назначенные в “Deny group” для группы к которой принадлежит напавление.
