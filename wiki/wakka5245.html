Режимы использования IP Studio AMBS



Режим «Парсинга»


В режиме «Парсинга» IP Studio AMBS будет через определенный интервал времени забирать с MVTS-сервера логи о прошедших звонках и загружать их в свою базу. По умолчанию, логи забираются каждый час, период времени указывается в планировщике сервера.
После загрузки данных в IP Studio AMBS, вы можете смотреть статистику прошедших звонков, если операторы и тарифы не были добавлены до загрузки звонков, их можно будет завести после, добавить им соответствующие тарифные планы и перерассчитать статистику по этим прошедшим звонкам с учетом цен операторов.


Это наиболее безопасный режим использования IP Studio AMBS, позволяющий ознакомится с системой, при этом не вызывая никаких серьезных изменений в работе. Поэтому, если к IP Studio AMBS подключается уже рабочий MVTS-сервер, всегда рекомендуется сначала использовать этот режим работы, пока не будут заведены все операторы и их тарифные планы.


Чтобы использовать этот режим, необходимо после добавления нового MVTS-сервера в базу системы (Equipments – MVTS servers) убедиться, что у этого сервера параметры “Enable Radius” и “Enable Accounting” установлены как – “No”. Если вы добавляете новый удаленный MVTS-сервер, сначала нужно выполнить все действия из раздела «Подключение удаленного MVTS-сервера» данного руководства. После этого нужно добавить пару следующих строк в планировщик MVTS-сервера (обычно это – “/etc/crontab”):


25 0 * * * root /usr/local/billing/bin/back.sh >> /usr/local/billing/logs/backup.err 2>&1
10 * * * * root /usr/local/billing/bin/load_cdr.pl >> /usr/local/billing/logs/loader.err 2>&1

Следует учитывать, что когда используются только удаленные MVTS-сервера, на сервере с IP Studio AMBS в планировщике должна присутствовать только одна строка, поскольку на нем не нужно запускать скрипт обработки логов:


25 0 * * * root /usr/local/billing/bin/back.sh >> /usr/local/billing/logs/backup.err 2>&1

И на каждом из удаленных MVTS-серверов в планировщик необходимо добавить две вышеуказанные строки.


После этого через заданный период времени запустится скрипт обработки логов, данные из логов будут загружены в систему. Также, вы можете проверить работоспособность, запустив данный скрипт вручную, выполнив следующую команду из консоли сервера:
/usr/local/billing/bin/load_cdr.pl


Если в процессе выполнения у вас появились ошибки, пожалуйста свяжитесь с техническим отделом разработчика.


Режим «Аккаунтинга»


В режиме «Аккаунтинга» с помощью системы AMBS вы сможете в он-лайн режиме получать информацию c MVTS-сервера о проходящих в данный момент через него звонкам, мониторить их и просматривать детальную статистику.


Для того, чтобы использовать систему в этом режиме, необходимо после добавления нового MVTS-сервера выделить этот сервер и нажать кнопку “Enable Accounting”.


После этого, нужно проверить, что в файле конфигурации “meraproxy.cfg” MVTS-сервера параметр “acct_enable” равен “1”.


После этого на странице “Calls online” (“Calls info” -> “Calls online”) у вас должны отображаться звонки, проходящие сейчас через MVTS-сервер.


Режим «Маршрутизации»


В этом режиме вы можете получать запросы от MVTS-сервера на разрешение прохождения звонка, и пути его следования. Будьте внимательны, если MVTS-сервер уже эксплуатируется и через него уже идут реальные звонки, то в этом режиме IP Studio AMBS сразу после установки запускать нельзя до тех пор, пока в нее не будет введена вся информация по вашим операторам, их ip-адресам и их тарифным планам. В ином случае, при переключении управления MVTS-сервера не до конца настроенной IP Studio AMBS, это может грозить полной остановкой рабочего процесса.


Если вы уверены, что потери звонков не произойдет, тогда чтобы включить этот режим, необходимо для выбранного MVTS-сервера в разделе серверов (Equipments – MVTS Servers) выделить этот сервер и нажать кнопки “Enable Accounting” и “Enable Radius”.
После этого нужно проверить, что в файле конфигурации “meraproxy.cfg” этого MVTS-сервера параметры “acct_enable” и “auth_enable” равны “1” и на странице “Calls online” (“Calls info” -> “Calls online”) отображаются звонки проходящие в текущий момент через MVTS-сервер.
