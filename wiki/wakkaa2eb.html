Секция сайтов оператора


Посмотреть пример подключения тарифного плана к сайту оператора



Сайт – это список шлюзов и назначенных им тарифных планов. Вся работа оператора строится «от» его сайтов. Каждый оператор изначально имеет по одному сайту, при необходимости количество сайтов может быть увеличено:



Система разделенных сайтов для каждого оператора была сделана для увеличения гибкости параметров соединения. Такая схема позволяет легко организовать работу оператора с отдельными тарифными планами для разных ip-адресов: например, оператор предоставляет два ip-адреса, по одному из которых он предоставляет направления среднего качества, а по второму, те же самые направления, но хорошего качества.


При создании нового оператора для него автоматически будет создан один пустой сайт с его именем. Каждый сайт позволяет закрепить за оператором по одном тарифному плану следующих типов:


ТипОписание
Terminate rateЗакрепляет выбранный тарифный план на терминацию

Originate rateЗакрепляет за оператором выбранный тарифный план на оригинацию

Saler rouming rateПозволяет выставить роуминговый терминационный тариф

Buyer rouming rateПозволяет выставить роуминговый оригинационный тариф

Operator clients rateЗдесь задается клиентский тариф, который данный оператор сможет в последствии регулировать сам для своих клиентов. Часто используется для переговорных пунктов

Internet terminate rateСекция для закрепления терминационных тарифных планов для интернет операторов. Используется для работы модуля NetFlow.

Internet originate rateСекция для закрепления оригинационных тарифных планов для интернет операторов. Используется для работы модуля NetFlow.

hunt modeРежим распределения нагрузки.
Может принимать следующие значения:
no resort(1) – функциональность отключена, и трафик направляется последовательно на все шлюзы оператора.
first to end(2) – адреса циклически меняются (первый адрес становится последним, второй – первым, и т.д.), смена шлюза происходит по завершении звонка.
by calls count(3) – адреса меняются в зависимости от загрузки шлюза (т.е. трафик направляется на наименее загруженный в данный момент времени адрес).
by calls count/capacity(4) – адреса меняются в зависимости от загрузки шлюза с учетом параметра capacity.
Значение по умолчанию – no resort

Кроме созданных вручную тарифных планов (в форме “Edit tariffs”), на оригинацию можно использовать служебную функцию «Себестоимость», это позволит звонить с этого сайта по всем доступным направлениям по себестоимости этих направлений. При использовании себестоимости нельзя выставить инвойс, поскольку отсутствует реальный тарифный план, поэтому не рекоммендуется использовать эту функцию для клиентов.


При создании сайта, за ним не закреплены никакие тарифные планы. Без тарифного плана через сайт звонки проходить не будут,нужно закрепить тарифный план и включить сервис (Enable service).


Примечание: При необходимости можно завести дополнительные сайты для оператора. На каждом из сайтов можно прописать добавочные оригинационные и терминационные тарифные планы.
Но не следует забывать, что шлюзы оператора закрепляются за определенным сайтом, поэтому для работы новых сайтов, за ними нужно закрепить новые шлюзы оператора (или старые, но уже с новыми трансляциями для входа и выхода трафика), чтобы система могла различать трафик оператора и разложить его по соответствующим сайтам оператора.

В форме сайтов оператора присутствуют следующие кнопки управления:


КнопкаНазначение
Позволяет добавить новый сайт оператору

Вызывает форму для редактирования параметров выбранного сайта


При нажатии любой из этих кнопок будет открыто окно для редактирования выбранного или добавления нового сайта:



В этой форме вы можете закреплять вышеописанные типы тарифных планов на сайте оператора. В форме редактирования сайта присутствуют следующие кнопки управления:


КнопкаНазначение
Сохраняет внесенные изменения и закрывает окно редактирования

Сбрасывает внесенные изменения и закрывает окно редактирования

Удаляет сайт оператора


Примечание:При наличии у оператора нескольких сайтов, система будет обрабатывать их в порядке, обратном порядку их добавления в систему, в первую очередь будут рассматриваться сайты добавленные в последнюю очередь.
