Рекомендации по безопасности серверов IP Studio AMBS


1. Закройте доступ к MySql из вне, оставьте только доступ с ваших ip-адресов: АМБС, удаленных мер и ваших рабочих, если вы непосредственно работаете с базой.
Для этого нужно консоль mysql выполнить следующие команды:

/usr/local/billing/mysql/bin/mysql
use mysql;
delete from user where Host='%';
GRANT ALL PRIVILEGES ON voip.* to voip@<your_ip_address> identified by '<your_password>';

где:
<your_ip_address> – на ваш ip-адрес (например самого сервера АМБС или удаленной меры).
<your_password> – на ваш новый пароль.

Например:

GRANT ALL PRIVILEGES ON voip.* to voip@192.168.1.1 identified by 'djbg123';

  • Если необходимо организовать доступ с нескольких ip-адресов, дайте несколько команд – GRANT с соответствующими ip-адресами.

  • Дайте право пользователю “voip” создавать файлы. Например:

GRANT FILE ON *.* TO voip@<ambs_ip_address> identified by 'djbg123';

где:
<ambs_ip_address> – ip-адрес АМБС сервера.


  • Дайте права пользователю voip, чтобы он мог устанавливать триггеры на своей базе:


grant SUPER on *.* to voip@localhost;
grant SUPER on *.* to voip@<ambs_ip_address>;


где:
<ambs_ip_address> – ip-адрес АМБС сервера.


  • В заключении, обновите привилегии, чтобы они вступили в работу:

flush privileges;

Примечания:
  • установленный ваш новый пароль должен совпадать с «your_password» паролем указанным в конфигурационных файлах системы.
По умолчанию в системе, для доступа к базе используется пароль: your_password – djbg123

Если вы решите заменить его, не забудьте изменить его в следующих конфигурационных файлах:

/usr/local/billing/etc/bill_util.cfg
/usr/local/billing/hosting/bill/.htaccess
/usr/local/billing/hosting/admin/.htaccess
/usr/local/billing/radius/radius.cfg
/usr/local/billing/radius/card.cfg (опционально, для пользователей карточного модуля)
/usr/local/billing/radius/dial.cfg (опционально, для пользователей диал-ап модуля)

2. Смените пароль для системного пользователя IP Studio AMBS – “ipstudio”. В консоли сервера выполните команду:

passwd ipstudio

и введите ваш новый пароль.

3. Смените пароль для пользователя “admin” в системе IP Studio AMBS, предварительно заведя нового пользователя в группе “Admins”.
В случае, если вы ошибетесь при смене пароля, у вас будет возможность поправить это.

4. Будьте внимательны при смене паролей, если после смены паролей у вас возникнут проблемы, свяжитесь с разработчиками.

Примечание: На АМБС-сервере у пользователя root должен быть доступ в mysql без пароля (update.pl выполняется из под root-а по техническим соображениям).
АМБС выполняет обновление своих скриптов и базы под этим пользователем, если root не будет иметь доступа в mysql или нужно будет вводить для него пароль, АМБС не сможет обновить свою базу.
