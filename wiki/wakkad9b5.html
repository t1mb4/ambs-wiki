Форма “Operator Route Parameters”. Ручная настройка маршрутизации


Форма предназначена для задания особых правил маршрутизации операторов.
Для каждого направления можно установить следующие параметры:

Для каждого направления оригинирующего оператора(ов) можно назначить:

  • Первоочередность операторов: Список из 3-х первых операторов, куда будет направлен трафик в первую очередь.
  • Назначение прямого маршрута: Безусловная маршрутизация трафика без контроля цены и качества на указанного оператора.
  • Лист запрещённых операторов: Список терминаторов, куда звонки не должны проходить.
  • Задать тип сортировки направлений: по цене, по качеству, по ASR и т.д,
  • Минимальный процент прибыли с направления.
  • Включить или отключить учет брекаутов для задаваемого правила.
  • Отключить прохождение трафика по более короткому коду, если такой имеется в тарифе оператора.
  • Закрыть выбранный код у оригинатора: Трафик этого оператора по этому коду приниматься не будет.

Оригинирующая форма имеет следующий вид:



Для каждого направления терминирующего оператора(ов) можно назначить:

  • Заблокировать направление или код для прохождения трафика как отдельный брекаут или полностью.
  • Задать допустимую ёмкость канала на направление.
  • Зарезервировать определённую ёмкость канала на направление за одной из тарифных групп (Route groups).

Терминирующая форма имеет вид:



Можно задавать правила для сайтов:
  • всех операторов (--All operators--)
  • списка выбранных операторов
  • одного конкретного оператора,

Примечание: В случае “All operators” будет создано одно «виртуальное правило» которое будет применяться для всех операторов выбранного типа.
Такое правило не будет дублироваться по всем сайтам операторов по отдельности.


Маршрутизацию можно задавать на:
  • отдельный код (single code) – конкретный набор цифр, с которого будет начинаться входящий номер.
  • группу кодов (destination) – набор кодов из базы данных АМБС.

Визуально форма состоит из двух частей. Первая (левая часть) отвечает за выбор сайтов операторов и направлений, для которых будет производиться ручная настройка маршрутизации. Вторая (правая часть) используется для задания параметров маршрутизации выбранным сайтам на указанные направления.

Левая часть формы состоит из следующих полей:

ПолеНазначение
Direction typeСлужит для выбора направлений, для которых будет производиться настройка маршрутизации.
Может принимать значения:
originate – будут производиться настройки для направлений оригинации трафика,
terminate – будут производиться настройки для направлений терминации трафика.
Значение по умолчанию – originate.

Select siteПозволяет выбрать сайт конкретного оператора или всех операторов в целом.
Может принимать значения:
All enabled – Позволяет просмотреть все активные на данный момент настройки по выбранному типу трафика,
All operators – Позволяет настроить маршрутизацию по выбранному коду(ам) всем операторам в целом,
Сайт оператора – Сайт конкретного оператора. Все настройки маршрутизации будут работать только для этого сайта.
Значение по умолчанию – All enabled.

Select countryФильтр по стране, показывает на экран направления только выбранной страны из базы системы.
Примечание: Можно производить поиск требуемого направления с использованием данного фильтра страны или сразу по нужному коду, если он известен заранее.

Select route byФильтр типа отображаемых направлений.
Может принимать значения:
Single code – Будут показаны только единичные коды.
Destinations – Будут показаны только группы направлений.
All – Будут показаны как единичные коды, так и группы кодов, удовлетворяющие условию выборки.
Значение по умолчанию – All.
Примечание 1: В некоторых ситуациях удобнее настраивать маршрутизацию на конкретный код, например закрыть у оператора терминацию на код 7495. В других удобнее использовать группы кодов, например, настроить для всех операторов первым выбором опеределенного оператора на все коды московских мобильных операторов.

Примечание 2:Следует обратить внимание на то, что “Single code” и “Destinations” с точки зрения системы это две разные функциональности. В случае использования типа “Single code” – система рассматривает набор цифр, составляющий заданный код, в то время как при использовании типа “Destinations” – система работает с конкретными id-номерами записей из базы направлений.

Select rows countКоличество строк направлений, показываемых за один раз на экране.
Может принимать значения: [10...100] или All
Значение по умолчанию – 20.

Filter optionsФильтр отображения направлений по статусу параметра маршрутизации.
Может принимать значения:
Select options – Неактивный фильтр. В таком режиме будут показаны все направления как с активными настройками маршрутизации, так и без них. Обычно используется для поиска направления для настройки маршрутизации.
Enabled only – Показывает только направления с активированными настройками маршрутизации. Обычно используется для быстрого нахождения уже используемых настроек, для их изменения или удаления.
Closed only – Показывает только направления которые были закрыты, со статусом – “Closed”.
Значение по умолчанию – Select option.

Filter codeФильтр по коду направления. На экран будут показаны все направления из базы системы, которые удовлетворяют условию данного поля.


Правая часть формы оригинации состоит из полей:

ПолеНазначение
Direct routeПрямой оператор. Позволяет задать одного прямого оператора для выбранного направления(й). Все другие операторы которые также предоставляют этот код направления, будут игнорироваться. Если звонок по этому направлению был отбит указанным оператором, этот звонок больше никуда не пройдет.
Важно: При использовании этого режима контроль над уровнем цен не используется, т.е. можно сработать в минус.

Route 1, 2, 3Первоочередные операторы, на которых будут направляться звонки по выбранному направлению(ям) в первую очередь. Если звонок был отбит этими операторами, дальше система будет пытаться отправить звонок по стандартной схеме, с учетом цен и других параметров сортировки.

Deny routeСписок запрещенных операторов. Для этого оригинирующего оператора по этому направлению(ям) операторы из этого списка являются запрещенными.
Примечание:Чтобы добавить оператора в «список запрещенных» нужно выбрать такого оператора из выпадающего списка “deny route” и нажать кнопку “set_route”.
Чтобы убрать оператора из «списка запрещенных» нужно в «Запрещенных операторах» убрать галочку с этого оператора и нажать кнопку “unset_route”.

Without breakoutsПозволяет использовать более длинные коды других операторов (если такие имеются) вперед первоочередных операторов. Если функция включена, и заданы конкретные операторы в полях (route 1,2,3), тогда перед направлением звонка на этих операторов система будет пытаться отправить звонок на других операторов, предоставляющих более длинные коды по данному направлению.

Пример:

У нас есть несколько терминирующих операторов предоставляющих следующие направления:
Оператора Б – направление с кодом “7”
Оператора В – направление с кодом “7495”
Оператора Г – направление с кодом “7499”

От «Оператора А» поступил звонок с номером – 7495.....
Для этого оператора установлены “route 1" -> «Оператор Б» и включена опция “Without breakouts”.

При такой схеме система сначала направит звонок на «Оператора В» и только потом на «Оператора Б».

Min percent of profitПараметр указывает минимальную прибыль с настраиваемого направления. Все терминирующие операторы, у которых разница между ценой оригинации выбранного оператора и их терминации будет меньше этого процента, будут исключены из списка терминирующих вариантов по этому направлению.

Route params enableАктивирует настройки. Это поле нужно активировать, если вы хотите использовать заданные вами настройки маршрутизации.
Если поле неактивно, настройки будут сохранены, но использоваться не будут.

Замечание:
Если настройки маршрутизации активны и используются, в листе направлений строка с кодом будет подсвечена зеленым цветом.
Если настройки присутствуют, но строка имеет стандартный серый цвет – настройки маршрутизации не активны.
Для их активации нужно выбрать это направление из списка и в поле “Route params enable” для него поставить галочку.

Sort typeТип сортировки списка терминационных направлений.
Может принимать следующие значения:
Price – Список доступных маршрутов будет сортироваться по минимальной цене,
ASR – Сортировка по лучшему показателю ASR,
ACD – Сортировка по лучшему показателю ACD,
Qi – Сортировка по общему индексу качества.
Значение по умолчанию – Price.

CloseЗакрыть направление. Вызовы выбранного оператора по закрытому коду приниматься не будут.

Proper rejectЗапрещает прохождение звонков на более короткие коды.
Например: Если данная функция настроена для кода 79, звонки настраиваемого оператора будут проходить на операторов, дающих только код 79 и более длинные. Все операторы, которые дают код – 7 – будут игнорироваться.

Special route %Заданный процент трафика будет уходить на указанный сайт. Позволяет выделить определенное количество трафика и направить его на отдельный сайт.

Only manual routesБудут работать только установленные 1,2,3 выборы. Другие маршруты для трафика по этому направлению предлагаться не будут.

Select mera serverПозволяет установить параметры маршрутизации для одной выбранной меры. Для других меры такое правило использоваться не будет.


В режиме терминации правая часть формы состоит из полей:

ПолеНазначение
CapacityПозволяет установить допустимую емкость на направление терминации

Guaranteed capacityГарантированная емкость для заданной группы. Позволяет зарезервировать определенную емкость направления терминатора за выбранной группой.

Without breakoutsИспользовать правило без учета брекаутов. Данное правило будет работать только для данного года.
Пример: Закрытие кода “74” без учета брекаутов при наличии у терминатора в тарифе кодов 74, 7495, 7499 заблокиреут только код 74.
Коды 7495 и 7499 по прежнему остануться в работе.

CloseЗакрыть выбранное направление для терминации.

Route parameters enableАктивирует настройки. Это поле всегда нужно включать, когда вы хотите запустить заданные вами настройки маршрутизации в работу.
Если поле включено настройки будут сохранены, но использоваться не будут.
Все активные правила маршрутизации выбеляются – зеленым цветом, не активные правила – серым.


Внизу формы присутствуют следующие кнопки управления:

КнопкаНазначение
Удаляет настройки параметров маршрутизации у кодов(групп направлений), выделенных в списке направлений.

Вызывает форму для добавления комментариев.

Сохраняет внесенные изменения в настройки.

Выводит на экран лист кодов(или групп кодов) с учетом заданных в форме параметров отображения.


Важно помнить следующие моменты при работе с данной формой:

1. Можно производить поиск требуемого направления с использованием фильтра страны или сразу по нужному коду, если он известен заранее.

2. Для сокращения списка показываемых кодов при поиске, удобнее сразу правильно указывать фильтр типа направлений, или вы будите настраивать маршрутизацию на конкретный код(single code) или на группу кодов(destinations).

3. При настройке “Direct route”, маршрутизация будет идти напрямую без контроля за ценой и статусом направления у терминирующего оператора. Т.е. если цена терминации выше цены оригинации или это направление закрыто у терминатора, звонок все равно будет смаршрутизирован выбранному оператору.

4. Если настройки маршрутизации активны и используются, в листе направлений строка с кодом будет подсвечена зеленым цветом.
Если настройки присутствуют, но строка имеет стандартный серый цвет – настройки маршрутизации не активны.

5. Коды для настройки маршрутизации берутся на прямую из общей базы кодов системы и не привязаны к тарифам операторов. Данная схема позволяет настроить маршрутизацию более гибко и точно.
