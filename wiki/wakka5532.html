Routs test. Тест доступных направлений


Форма тестов направлений позволяет просмотреть доступные оператору (оригинатору) направления по любому номеру.
Можно увидеть, на какие ID-маршрутов пойдет звонок оператора с учетом всех правил маршрутизации и его цены на данное направление.
Внешне форма выглядит следующим образом:



Описание полей формы:

ПолеНазначение
Select siteПоле для выбора сайта оригинирующего оператора, для которого будет строиться список доступных ему маршрутов.

Source ipАдрес исходящего шлюза, с которого поступает звонок.
Необязательное поле, можно оставлять пустым.

Select mera serversАдрес MVTS-сервера, через который будет проходить звонок.

Destination numberНомер (или код), по которому требуется просмотреть список доступных для терминации маршрутов.

Show big pricesПараметр для просмотра более дорогих роутов, на которые звонок не может пройти из-за установленной цены на оригинацию.

Closed routes (y/n)Параметр для просмотра закрытых направлений операторов.
Существующих направлений, на которые звонок может пройти, но направления закрыты по каким-то причинам (не работающий код, неправильная цена и т. п.)

Check number lengthПараметр для проверки указанного номера в поле «Destination number» на правильность, если в системе используется фильтрация на длину номера.

Route limitКоличество строк выводимых на экран.

Show routeКнопка запуска выборки. Передает на сервер заданные параметры и выдает результат выборки на дисплей ПК.


Для того чтобы воспользоваться данной формой достаточно:

  • В поле «Select site» необходимо выбрать оригинирующего оператора;
  • В поле «Select MERA servers» следует выбрать один из MVTS-серверов, прописанных в системе;
  • В поле «Enter destination number» необходимо ввести начальные цифры кода направления;
  • Нажать кнопку «Show route».

В результате получим таблицу следующего вида:




Под формой показана строка с информацией по направлению из тарифного плана оригинатора, которую система использовала для построения листа доступных маршрутов.
В строке видно:
  • Цена оригинации – 0.0052;
  • Код оригинации – 7495;
  • Процент прибыли со звонков оригинирующего оператора – 0.0010;
  • Максимальная доступная цена терминации (с учетом процента прибыли) – 0.00519.

Далее в таблице приведены доступные направления терминации. Система покажет список с учетом всех установленных для данного направления оператора правил маршрутизации и сортировки.
Например:
если у оригинирующего оператора указано сортировать доступные ему направления по цене, то направления будут строиться с учетом цены, т.е. с минимальной цены до максимальной.


В таблице доступных направлений различными цветами подсвечена доступность операторов и правила маршрутизации:

ЦветНазначение
ID-маршрута, поставленного вручную первым выбором.
Настройки ручной маршрутизации производятся в форме «Operator route parameters».

ID-маршрута, поставленного вручную вторым выбором.
Настройки ручной маршрутизации производятся в форме «Operator route parameters».

ID-маршрута, поставленного вручную третьим выбором.
Настройки ручной маршрутизации производятся в форме «Operator route parameters».

Дорогие маршруты. ID-маршрутов, на которые данный оригинатор пройти не может из-за установленной цены на оригинацию.

Закрытые маршруты. ID-маршрутов, закрытые вручную.

Направления, находящиеся в одной группе с направлением оригинатора.


Таблица со списком доступных маршрутов состоит из следующих полей:

ПолеНазначение
NumПорядковый номер по списку.

IDИдентификатор маршрута.

Proxy typeСтатус режима проксирования:
1 – проксирование используется;
0 – проксирование не используется.

ASRВеличина коэффициента дозвона (ASR=состоявшиеся звонки/попытки * 100%).

ACDСредняя продолжительность разговора в минутах.

Short callsПроцент коротких звонков, у которых продолжительность меньше 10 секунд.

All callsОбщее количество прошедших звонков.

Complete callsКоличество звонков с ненулевой длительностью.

QiИндекс качества маршрута по выбранному коду.
Выбранный код из тарифного плана маршрутов показывается в поле «Code».

Dst number transНомера на выходе, такой каким он уйдет к терминирующему оператору.

CodeКод из тарифного плана маршрутов, который система нашла по запросу.


Примечание: Если на проверяемое направление выводится надпись «Codes not found» – это означает, что на проверяемое направление нет доступных маршрутов.
