Форма “Autoclose”. Форма алертинга и автозакрытия.


Позволяет задать для сайтов операторов минимальные пороговые показатели качества на направления и отправлять уведомления при падении качества или автоматически закрывать плохие направления.



Форма состоит из следующих полей:

ПолеНазначение
Select siteВыбор сайта оператора для которого добавляется правило.
Может принимать значения:
All operators – Все операторы, правило будет применяться для всех операторов в АМБС.
All enabled – Все активные правила. Используется для поиска нужно правила.
Selected operators – Выбранные сайты операторов.
Значение по умолчанию: All operators

Select route byТип выбираемых маршрутов.
Может принимать значения:
Single code – Будут показаны только единичные коды.
Destinations – Будут показаны только группы направлений.
All – Будут показаны и коды и группы направлений.
Значение по умолчанию: Single code

Filter optionsФильтр отображения направлений по статусу параметра маршрутизации.
Может принимать значения:
Select options – Неактивный фильтр. В таком режиме будут показаны все направления как с активными настройками маршрутизации, так и без них. Обычно используется для поиска направления для настройки маршрутизации.
Enabled only – Показывает только направления с активированными настройками маршрутизации. Обычно используется для быстрого нахождения уже используемых настроек, для их изменения или удаления.
Closed only – Показывает только направления которые были закрыты, со статусом – “Closed”.
Значение по умолчанию – Select option.

Select userФильтр активных правил по пользователю, установившему правило.

Select countryФильтр по стране, показывает на экран направления только выбранной страны.

Filter codeФильтр по коду направления. На экран будут показаны все направления из базы системы, которые удовлетворяют условию данного поля.

Show current statisticПоказывать в активных правилах текущую статистику по направлению.

Period rangeПериод времени в минутах через который выполнять проверки правил.

Minimum calls countМинимальное количество звонков на направление которое можно считать достаточным для контрольного замера качества.
Если звонков за период будет меньше, такой период будет игнорироваться.

Uniqe numbers onlyСчитать только уникальные номера за период, исключить повторение номеров.

Never closeНе закрывать направление, если показатели качества меньше пороговых значений, только информировать.

Close time countКоличество этапов закрытия направления. Последнее закрытие будет полное, после этого направление нужно будет разблокировать вручную, форме “Route Paramaters”.

Close hoursПериод в часах, на который будет закрываться направление, если показатели качества ниже пороговых.
По истечение этого периода времени, направление будет открыто.

Close hours additionalyДобавочный период времени в часах. Будет добавляться к параметру “Close hours” для каждого последующего этапа закрытия.

Пример: Close count – 3, Close hours – 6, Additional – 6. В этом случае направление будет закрыто 3 раза, первый раз на 6 часов, второй на 12, 3й – полностью.

Mail onlyНе закрывать направление, только информировать по почте.

Send mail to our managerПозволяет отправить письмо менеджеру ведущему оператора уведомление о падении качества.

Send mail to operatorПозволяет отправить письмо оператору на его адреса (коммерческий и технический) о падении качества.

Send mail to our technikПозволяет отправить письмо технику о падении качества у оператора.

Additional mailsДополнительные адреса, куда нужно отправить уведомление.

No breakoutsУчет брекаутов при закрытии.
Может принимать следующие значения:
the code including longer – Закрытать указанный код без учета более длинных брекаутов (полное закрытие).
the code only – Закрывать только заданный код, более длинные коды пропускать (если такие есть в тарифе оператора).
use the rule for all breakouts – Закрыть все брекауты, указанный и более длинные.
Значение по умолчанию – use the rule for all breakouts

Min ASRМинимальный порог ASR, ниже которого считать качество неудовлетворительным.

Min ACDМинимальный порог ACD, ниже которого считать качество неудовлетворительным.

Max Short CallsМаксимальный процент коротких звонков, выше которого считать качество неудовлетворительным.

Min Complite CallsМинимальный процент завершенных звонков (звонки, по которым был разговор), ниже которого считать качество неудовлетворительным.

AND-ORФормат использования пороговых значений «И»-»ИЛИ» при срабатывании правил.
AND – Если все из показателей не удовлетворяют качеству, считать направление неудовлетворительным и применить правило.
OR – Если любой из показателей не удовлетворяет качеству, считать направление неудовлетворительным и применить правило.
Значение по умолчанию: OR
