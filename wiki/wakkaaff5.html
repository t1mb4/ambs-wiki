Роуминг. Организация транзитного и частного роуминга

Организация транзитного роуминга


Система IP Studio AMBS позволяет осуществить транзит выбранного номера через систему с одного оператора на другого.
Организация транзитного модуля в системе IP Studio AMBS представлена следующим образом:


1. Завести роуминговый номер в базе системы, как роуминговый номер, в форме «Edit destination».
При добавлении роумингового номера следует учитывать условия:

  • Заполненными должны быть только поля «Destination» и «Code»;
Примечание: в поле «Code» указывается непосредственно сам роуминговый номер.

  • Поле «Country code» можно оставить пустым;
  • Тип номера нужно указать – «Roaming».


2. Создать роуминговый тарифный план для «Продавца номера» (Оператор 1), добавить в тариф этот номер.



Примечание: Если тарификация будет поминутно, можно указать цену или оставить цену нулевой.
Номер нужно открыть, убрать галочку «Closed» после его добавления в тарифный план.

3. Создать роуминговый тарифный план для «Покупателя номера» (Оператор 2), добавить в него этот роуминговый номер.



4. Проверить, что в обычном оригинационном тарифе «Продавца номера» (Оператор 1) обязательно присутствует для оригинации код направления, которому принадлежит этот роуминговый номер (в нашем случае, в оригинационном тарифе должно присутствовать направление – «123»), иначе биллинг не примет номер от этого оператора.


Примечание: Если у «Продавца номера» нет обычного оригинационного тарифа, его следует добавить для работы роумингового номера.

5. У «Продавца номера» (Оператор 1) на его сайте следует закрепить роуминговый тариф продажи (в нашем случае, «Oper1_roam_sell») в поле «Saler roaming rate».



6. У «Покупателя номера» (Оператор 2) на его сайте следует закрепить роуминговый тариф покупки (в нашем случае, «Oper2_roam_buy») в поле «Buyer roaming rate».



7. Проверить в форме «Route test», что этот номер проходит на нужного оператора, в нашем случае «Оператор 2».


8. Сделать тестовый звонок по не-VoIP-ным линиям и проверить, как пошел звонок. Номер должен приходить от Оператора 1 и уходить на Оператора 2.

Примечание: Детальную статистику по прошедшим звонкам для транзитного роуминга можно просмотреть в форме «Call Details»

Роуминг для частных лиц


Частный роуминг в системе IP Studio AMBS организован следующим образом:

1. В форме «Edit Destinations» завести роуминговый номер (как это написано выше).


2. В форме «Edit tariffs» подготовить обычный тарифный план с направлениями (оригинационный), куда планируется производить переадресацию.


3. В форме «Roaming» (Service – Roaming) создать нового роумингового клиента.


4. Далее, в роуминговой форме для нового клиента назначить переадресацию с выбранного роумингового номера на номер клиента, закрепив за этим клиентом тарифный план.


В результате, звонок приходящий в систему с выбранного роумингового номера, будет направляться на номер клиента по доступным VoIP-линиям.


Основная форма для работы по роумингу с частными лицами, форма «Roaming» в разделе «Services»:



Левая часть формы служит для выполнения операций по работе с роуминг-клиентами, правая для выбора временного интервала за который будет выводиться статистика по прошедшим роуминг-звонкам.


Описание полей формы:


ПолеНазначение
Login Позволяет выбрать для просмотра конкретного роуминг-клиента.
Значение по умолчанию – All clients.

Time periodПериод времени, за который будут показаны звонки на роуминг-номера.
Может принимать следующие значения:
Today – сегодня;
Yesterday – вчера;
3 days – три дня;
7 days – семь дней, не включая текущий день;
Month – текущий месяц;
Last month – предыдущий месяц;
Period – период, заданный вручную полями From-To.
Значение по умолчанию: Today.


В форме присутствуют следующие функциональные кнопки:


КнопкаНазначение
Add private client. Добавляет роуминг-клиента в систему.

Find roaming user. Отображает реквизиты зарегистрированных в системе роуминг-клиентов.
Кнопка работает только при условии – All clients.

Edit client info. Вызывает форму для редактирования информации о клиенте.
Show calls. Выводит статистику по прошедшим звонкам.

Show pays. Показывает историю платежей на баланс клиента.

Show forward history. Показывает историю переадресации роуминг-номеров на номера клиента.

Пример организации частного роуминга


1. Завести роуминговый номер в базе системы, как роуминговый номер, в форме «Edit destination».
При добавлении роумингового номера следует учитывать условия:

  • Заполненными должны быть только поля «Destination» и «Code»;
Примечание: в поле «Code» указывается непосредственно сам роуминговый номер.

  • Поле «Country code» оставить пустым;
  • Тип номера указать – «Roaming».


2. В форме «Edit tariffs» cоздать стандартный оригинационный тарифный план.


3. В форме «Roaming» завести роуминг-клиента и прикрепить за ним тарифный план, воспользовавшись кнопкой «Add private client».



4. После заведения роуминг-клиента, следует в поле «Login» (1) выбрать созданного клиента, добавить денежную сумму на баланс (2) и назначить переадресацию роумингового номера на номер клиента (3).
