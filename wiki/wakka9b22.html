Форма “Card statistic”


В этой форме пользователь может получить полную информацию по любой карте, какие звонки были совершены, статус карты, оплаты по карте, если карта кредитная и другую информацию. Общий вид формы статистики по карте показан на следующем рисунке:



Описание полей формы:


ПолеНазначение
FilterПозволяет найти карточку по заданным параметрам:
Card number – поиск по названию карточки;
PIN code – поиск по пин-коду карточки;
VoIP – поиск по типу карточки VoIP;
Not zero calls – будут показаны только ненулевые звонки по выбранной карточке.

Time period Период времени, за который будут показаны звонки.
Может принимать следующие значения:
Today – сегодня;
Yesterday – вчера;
3 days – три дня;
7 days – семь дней, не включая текущий день;
Month – текущий месяц;
Last month – предыдущий месяц;
Period – период, заданный вручную полями From-To.
Значение по умолчанию – Today.


В форме присутствуют следующие кнопки управления:


КнопкаНазначение
Find. Осуществляет поиск карты по указанным параметрам и выводит статистику звонков по этой карте.

Add pay. Позволяет добавить платеж на баланс карточек.

Show pays. Показывает историю платежей на балансе карточки.


Статус карты и статистика по звонкам прошедшим через выбранную вами карту, будут выводиться в виде двух таблиц, которые представлены на следующем рисунке:



В верхней форме возможно редактирование уже существующей карты, описание полей формы:


ПолеНазначение
Personal card Позволяет выбранную карточку закрепить за каким-либо абонентом.

Active dateДата и время активации карты.

Close dateДата, когда карта будет автоматически закрыта, если карта имеет ограниченный срок действия. Если период работы не ограничен, это поле будет иметь статус «not established».

Current amountТекущий баланс. Активный баланс на текущий момент времени доступный для разговоров.

Min. amountМинимальный баланс, по достижению которого карта блокируется.

Current statusСтатус карты. Можно принудительно закрыть или повторно активировать карту. Что бы сохранить внесенные изменения, следует нажать кнопку «Edit card».

Change lotПозволяет перенести карту в состав другого лота. Что бы сохранить внесенные изменения, следует нажать кнопку «Edit card».

Authorization typeТип авторизации карты. При необходимости его можно изменить.

CategoryТип карты. Карта бывает следующих двух типов:
Debit – Уже оплаченные карты. Пользователь может звонить по таким картам до тех пор, пока баланс не дойдет до значения «Min. amount». Рассматривается только положительный баланс;
Credit – Кредитные карты. Баланс таких карт может уходить в минус.

Max. sessions Количество одновременных звонков по одному и тому же пин-коду.

ServiceТип создаваемых карт. При условии что установлен диал-ап модуль IP Studio AMBS, может принимать следующие значения:
VoIP – создать VoIP-карточку;
Dial-up (time) – создать карточку с типом Dial-up;
VoIP+Dial-up (time) – создать единую карточку и для VoIP, и для Dial-up.

LanguageВыбор языка в IVR.

ANI listЛист с телефонными номерами, которые будут использоваться во время авторизации карты.


В нижней таблице будут показаны все прошедшие и непрошедшие звонки по выбранной карточке, описание полей таблицы:


ПолеНазначение
Call timeДата и время когда проходил звонок.

DNISНомер доступа на карточную платформу.

ANIНомер звонящего абонента.

Dial numberНомер вызываемого абонента.

TariffЦена направления, по которому прошел звонок.

Call duration (sec.)Продолжительность разговора.

Rounded duration (sec.)Продолжительность разговора с учетом округления.

Call priceСтоимость прошедшего звонка.

Return codeКод Q931, с которым закончился звонок.

В форме «Card statistic» пользователь, при возникновении любых технических вопросов по работе конкретной карты или по ее состоянию, может получить исчерпывающую информацию.
