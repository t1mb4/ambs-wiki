Edit tariffs. Просмотр тарифного плана


В этой форме возможно просмотреть тарифный план оператора (ов), которого (ых) курирует дилер.



Форма состоит из следующих полей:


ПолеНазначение
Select rate typeТип тарифных планов (далее просто тариф). Необязательное поле.
Может принимать следующие значения:
Roaming – Роуминговые тарифы;
Originate – Оригинационные тарифы;
Terminate – Терминационные тарифы;
Cards – Карточные тарифы;
Clients – Клиентские тарифы.

Select rateПоле для выбора одного из уже существующих в системе тарифных планов, закрепленных за доступными операторами для дилера. После выбора одного из них, лист направлений выбранного тарифного плана будет показан ниже в виде списка и доступен для просмотра.

Rate nameСтрока для ввода имени для нового тарифного плана или для изменения имени выбранного тарифа на новое имя. Для дилерской группы это поле не функционирует.

Select currencyСписок используемых в системе валют. Служит для выбора валюты, в которой будут показаны цены на направления.
Может принимать значения любой из валют заведенных в системе.
Значение по умолчанию – USD.

Sub payАбонентская плата для роуминговых тарифов. Указанная сумма будет ежедневно вычитаться из баланса пользователей. Для дилерской группы это поле не функционирует.

Select countryФильтр по стране. Позволяет просмотреть все направления только выбранной страны. Поле работает сразу с двумя частями формы: фильтрует направления уже созданного тарифного плана и выводит все направления этой страны в правую часть формы для добавления одного из них в тарифный план.

Select typeФильтр по типу направлений (пропер, сотовые, городские, региональные и т. д.).
Поле работает с двумя частями формы: фильтрует направления уже созданного тарифного плана и выводит соответствующие направления в правую часть формы.

Rates list rows countКоличество строк тарифного плана, которые будут показаны на экране за один раз.
Может принимать следующие значения: 10, 20, 30, 50, 100 или All.
Значение по умолчанию – 20.

Full codes (y/n)Тип отображения кодов в направлениях в листе тарифного плана. Если галочка стоит, все направления будут показаны с полным списком кодов входящих в состав каждой записи. Если галочка убрана, все записи будут показаны в сокращенном виде – будут показаны только первые три кода направлений в каждой записи.

Dest numberФильтр цифр кода направления. В листе направлений тарифного плана будут показаны только те направления, код которых начинается (или соответствует) с указанных цифр.

Filter date (yyyy-mm-dd)Фильтр даты начала работы направлений в тарифном плане. В тарифном плане будут показаны только те направления, у которых поле «Start date» совпадает со значением, указанным в данном поле. Указываемая дата должна иметь формат: ГГГГ-ММ-ДД.

Status filterФильтр просмотра направлений тарифного плана.
Может принимать одно из двух значений:
Current – Активные направления. Будут показаны только активные на данный момент времени направления. Под активными направлениями понимаются все направления, у которых текущая дата попадает в период между «Start date» и «Stop date» этих направлений;
History – Полный список направлений. Будут показаны все направления, включая записи, которые работали в прошлом, и будут работать в будущем.
Значение по умолчанию – current.

Rates status filterФильтр статуса направлений. Будут показаны только направления с выбранным статусом.
Может принимать следующие значения:
All rates – Полный список направлений;
Closed only – Только закрытые направления;
Opened only – Только открытые направления.
Значение по умолчанию – All rates.

Time zones filterФильтр по временным периодам. В выбранном тарифном плане будут показаны только те направления, которые активны в выбранный временной период.
Может принимать значение любого периода времени, заведенного в системе.
Значение по умолчанию – Any Time Zones (все периоды).

Excel export filterСписок с вариантами отображения кодов в направлениях тарифного плана при его экспорте в CSV-файл.
Может принимать следующие значения:
Format_string – Система будет экспортировать все записи направлений с учетом регулярных выражений системы (коды направлений будут записаны в универсальной форме с сокращениями);
Comma_delimited – Все коды направлений будут записаны через запятую;
By_codes – Все коды будут записаны отдельно. Если одно направление состоит из группы кодов, такая строка будет разделена на несколько строк, по одному коду в каждой.
Значение по умолчанию – Format_string.


В форме присутствуют следующие кнопки управления:


КнопкаНазначение
Rename destinationsПереименовывать направления при их загрузке в базу. Если эта опция активна, система будет загружать коды с их названиями из файла, если в системе уже присутствовали эти коды, их названия будут перезаписаны на новые. Для дилерской группы эта функция не работает.

Позволяет добавить комментарий по оператору.

Filter (правая часть экрана)Фильтр по существующим в системе направлениям. Используется для добавления единичного направления в тарифный план. Для дилерской группы эта функция не работает.


После выбора тарифного плана из списка, будет показан лист с его направлениями с учетом всех установленных фильтров:



В данном случае был использован фильтр активных направлений – «current», этот фильтр используется по умолчанию, чтобы не загружать экран лишней информацией – направлениями, которые в данный момент не активны.


Если изменить значение фильтра активности на «history», система покажет полный список направлений в тарифном плане: направления, которые работали раньше, работают сейчас и будут работать в будущем. Например, предположим, что сегодня – 12–02–2007, тогда при просмотре выше показанного тарифного плана с использованием фильтра активности «history», мы получим следующий вид:



Как видно из рисунка, в данном случае направлений больше. Следует обратить внимание на даты активности направлений. Разберем их по порядку:


  • Kyrgyzstan Bishkek (996312) – Направление закрыто галочкой Closed и имеет цену 0 – так добавляется в тарифный план единичное направление. После добавления такое направление следует отредактировать с учетом его цены и периода активности. На данный момент на это направление трафик проходить не будет;
Примечание: Для дилерской группы редактирование направления недоступно.

  • Russia Moscow (7495) – Для этого направления присутствуют две записи: первая с периодом работы «2006–02–10» – «2007–06–10» будет активна на данный момент времени. Текущая дата 2007–02–12 попадает в интервал между «Start» и «Stop»-датами работы направления. Вторая запись «2007–06–10» – «9999–01–01» с новой ценой будет активна в будущем, с 2007–06–10;

  • Russia st.Petersburg(7812) – Также имеем две записи: первая с периодом работы «2005–01–01» – «2006–02–10» – прошедший период работы, по которому направление работало раньше со старой ценой. Вторая запись «2007–02–10» – «9999–01–01» активна на данный момент времени – запись с текущей ценой.
Примечание: Дата вида «9999–01–01» означает в системе открытую дату, без конкретного срока завершения работы.

При необходимости выбранный тарифный план можно сохранить в файл формата .csv, воспользовавшись кнопкой «Excel version» внизу листа направлений. При этом все направления в файле будут записаны с учетом выбранного вами формата записи кодов: format_string, comma_delimited или by codes.
