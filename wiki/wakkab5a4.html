Пример заведения одного ip-адреса несколько раз


Разберем следующий пример:

Терминирующий оператор просит направлять на него весь трафик без префикса, кроме Вьетнама.
Вьетнам должен идти с префиксом – 100. Оператор имеет один шлюз.


Для решения такой задачи нам потребуется:
– завести для этого оператора два тарифных плана (весь мир и Вьетнам),
– прописать шлюз оператора два раза (с префиксом – 100 и без префикса).


Поскольку на одном сайте оператора мы можем закрепить только один терминирующий тарифный план, нам потребуется завести дополнительный сайт для терминации Вьетнама.


Для выполнения этих условий нужно выполнить следующие действия:

1. Прежде всего, для этого оператора нужно создать два тарифных плана.
В первый тарифный план (например: Test_oper_term) нужно собрать все направления которые предлагает оператор, кроме Вьетнама.
Во второй тарифный план (например: Test_Oper1_Vt_term) соберем все направления по Вьетнаму.

2. Выбираем этого оператора в разделе “Edit operator”. Изначально, у него есть только один сайт (например: Test_oper_1).


Рисунок 1.


Для решения нашей задачи нам потребуется добавить ему еще один, дополнительный сайт. На панели сайтов оператора нажимаем кнопку , в окне нового сайта указываем имя для нашего нового сайта (например: Test_oper_1_Vt) и сохраняем этот новый сайт, нажав на кнопку .

3. Выбираем первый сайт оператора (Test_oper_1), редактируем сайт и добавляем к нему тарифный план для всего мира, трафик которого должен будет идти без префикса – Test_oper_term.

4. В секцию шлюзов на первом сайте добавляем ip-адрес шлюза оператора (по правилам заведения нового шлюза), поля “Dst_translate” и “In_dst_translate” оставляем пустыми.

5. Выбираем второй сайт оператора (Test_oper_1_Vt), редактируем сайт и добавляем к нему тариф с направлениями Вьетнама (Test_Oper1_Vt_term), трафик которого должен идти с префиксом – 100.

6. Переходим на первый сайт оператора. Находим шлюз, который мы добавляли в пункте 4 выше и выполняем следующие действия:

  • нажимаем кнопку чтобы продублировать этот шлюз,

  • в открывшемся окне параметров шлюза, проверяем, что все параметры полностью совпадают с параметрами первого шлюза (GW ID должен быть таким же, как и на первом сайте шлюза),

  • добавляем этому новому адресу в поле “Dst_translate” значение: [0–9]*/100& ,

  • в поле “In_dst_translate” добавляем значение: 100|.*/\2 ,

  • в поле “Change site” ставим галочку и выбираем из списка второй сайт этого же оператора (Test_oper_1_Vt),

  • нажимаем кнопку чтобы сохранить дублируемый шлюз на втором сайте оператора.

Важно: Один и тот же ip-адрес можно использовать в системе несколько раз, только если поля оба поля “Dst_translate" и «In_dst_translate” у этих одинаковых адресов будут разными.


Рисунок 2.


7. Переходим на второй сайт оператора и проверяем добавленный шлюз. Вы должны получить результат аналогичный показанному на рисунке 2.

8. Сделайте пару тестовых звонков через этого оператора. В разделе “Call details” проверьте поле “dst_number_out” ваших тестовых звонков. Номер должен уходить с префиксом 100 на Вьетнам и без него по всем остальным направлениям.
