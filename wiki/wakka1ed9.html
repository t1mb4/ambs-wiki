Форма “Edit Destinations”


В данной форме хранится база кодов направлений, которые использует система в процессе своей работы. Любое направление может быть записано в системе в виде единичного кода или группы кодов, с использованием регулярных выражений системы. Новые направления можно подгружать в систему при загрузке тарифов из CSV файла или заводить вручную в этом разделе.

Для того чтобы оператору назначить направление, чтобы он мог оригинировать или терминировать по нему, предварительно это направление нужно завести в системе, указать его параметры, к какой стране принадлежит это направление, из каких кодов оно состоит и т.д. После того, как направление будет добавлено, его можно добавлять в тариф оператора для работы. При импорте тарифов из файла все эти операции делаются автоматически.

Внешне форма выглядит следующим образом:



Форма состоит из двух частей. В верхней расположены фильтры, позволяющие быстро выбрать любые направления из системы не просматривая все коды в целом:

ФильтрНазначение
Destination view filterРежим просмотра направлений.
Может принимать значения:
By codes – Показывает коды без использования регулярных выражений системы. Если направление содержит группу кодов, они будут записаны через запятую.
By rates orig – Показывает направления с их раскладкой по тарифам оригинирующих операторов. В каком тарифе закреплен каждый код.
By rates term – Показывает направления с их раскладкой по тарифам терминирующих операторов. В каком тарифе закреплен каждый код.
Simple – Базовый вид направлений, показывается название направления и закрепленные за ним коды. Каждое направление показывается в виде отдельной строки.
Значение по умолчанию – simple.

Country filterПозволяет выбрать конкретную страну из списка заведенных в системе кодов стран для просмотра кодов, принадлежащих только этой стране.

City filterЕсли используется код страны, позволяет выбрать конкретный город из списка заведенных в системе кодов городов, принадлежащих выбранной стране. Показывает только коды, принадлежащие выбранному городу.

Destinations type filterФильтр по типу направлений. Proper, cell, region, dnis, roaming и т.д.
Будут показаны направления только с выбранным типом.

Max rows per page filterКоличество строк в листе направлений показываемых за раз на экран.


Нижняя часть служит для поиска конкретного направления по его коду или по его шаблону.
Под шаблоном понимается поиск по части названия страны или города, к которым принадлежит искомый код.
Особо следует отметить возможность использования расширенного поиска с использованием символа %. Данный символ означает любое продолжение строки, т.е. режим поиска расширяется кроме поиска указанного кода или шаблона кода, до поиска такого кода плюс любого более длинного кода имеющего такую же основу. Например:

1. Найти код 7495

Search by code: 7495

2. Найти все коды, начинающиеся с 79:

Search by code: 79%

3. Найти все коды заканчивающиеся на 312:

Search by code: %312

4. Найти все коды со словом Moscow:

Search by template: %Moscow%

Примечание: Символ % можно использовать как в начале строки поиска, так и в ее конце.

Также в форме присутствуют следующие кнопки управления:

КнопкаНазначение
Открывает форму для добавления нового направления в базу.

Запускает процесс поиска по заданным параметрам.



Направления, заведенные в системе, при просмотре будут иметь следующий вид:



Из рисунка видно, что один и тот же код может быть заведен в базе несколько раз в различных группах кодов.
При выводе списка направлений на экран внизу списка показывается меню с кнопками по работе со списками кодов. Меню состоит из следующих кнопок:

КнопкаНазначение
Delete selected destinationsУдаляет из показанного на экране листа направлений все активные направления. Направления активируются галочкой в поле “Check”.

Change destination in rate plansПозволяет произвести замену одного кода (группы кодов) на другую во всех тарифных планах системы за раз. Например, при смене кода направления.

Update all codesПерепроверяет все коды в базе на наличие ошибочных кодов, лишних пробелов в группах кодов,
или запятых (т.н. «пустые коды»).

Delete unusableУдаляет из базы системы все неиспользуемые (не закрепленные в тарифах) коды.
Функция полезна, если вы постоянно работаете с тарифами типа A-Z.

Delete duplicateПроверяет базу кодов и удаляет все повторяющиеся коды.
